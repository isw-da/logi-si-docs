# Save Analysis Templates

You can save the result of your data analysis during a visual analysis session as an analysis template in the server resource tree.

To save the result, select the **Save** button ![Save button](https://devnet.logianalytics.com/hc/article_attachments/4420461454871/btn_save.gif) on the toolbar. If the analysis template is newly created and has not yet been saved, Server displays the [Save As](https://devnet.logianalytics.com/hc/en-us/articles/4405683785751-Save-As-Dialog-Box-Properties) dialog box.

![Save As dialog](https://devnet.logianalytics.com/hc/article_attachments/4420447261591/svas.gif)

1. In the Save In drop-down list, browse to the folder where you want to save the analysis template in the server resource tree. You can use the button ![Up Level button](https://devnet.logianalytics.com/hc/article_attachments/4420453485079/btn_dshbrd_up.gif) to return to the parent folder.
   You cannot use the root folder to store resources.

   The resource table shows the analysis templates in the current directory. You can select the column names to change the order of the analysis templates in the table list.
2. In the File Name box, type the name of the analysis template or use the default name, without suffix.
3. Select **OK** to save the analysis template. The saved template is linked to its original catalog.

To save a copy of an analysis template, select ![Save As button](https://devnet.logianalytics.com/hc/article_attachments/4420447052311/btn_saveas.gif) on the toolbar or select ![Menu button](https://devnet.logianalytics.com/hc/article_attachments/4420447359255/btn_va_menu.gif) > **Save As** to show the Save As dialog box, and then do as above. If you are saving to an existing file, Server displays a Confirm dialog box asking whether you want to replace the file or [save a new version into the file](https://devnet.logianalytics.com/hc/en-us/articles/4405683940759-Creating-Versions#Save).

After saving the analysis template into the server resource tree, you can browse to its directory in the server resource tree on the Server Console and run it directly.

![Note icon](https://devnet.logianalytics.com/hc/article_attachments/4420447034135/noteicon.jpg) You will not be able to save resources to some locations if you do not have the Write permission.
