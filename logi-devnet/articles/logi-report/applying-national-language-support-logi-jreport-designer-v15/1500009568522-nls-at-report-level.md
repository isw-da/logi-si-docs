# NLS at Report Level

The NLS feature is supported for the language-dependent text in the label objects, DBFields, parameters, formulas, summaries, some special fields (Print Date, Fetch Date, Modified Date, Page N of M, Global Page N of M, group by field), table/view/synonym columns, imported SQL columns and so on. Logi JReport Designer provides with two editors to edit NLS:

* **Data Mapping Editor**  
   The Data Mapping Editor is used to map data values of resources in a catalog to a specific language, so that when you use the resources in reports, you can choose to display their values in the mapped language.
* **NLS Editor**  
   The NLS Editor is used to edit the language information of a report and the current catalog and stores the information in a NLS property file.
