# Saving Analysis Templates

The result of your data analysis during a visual analysis session can be saved as an analysis template into the My Reports or Public Reports folder in the server resource tree. To do this, select the **Save** button ![Save button](https://devnet.logianalytics.com/hc/article_attachments/4404926628119/btn_save.gif) on the toolbar. If the analysis template is newly created and has not yet been saved, the [Save As](https://devnet.logianalytics.com/hc/en-us/articles/1500009647002-Save-As) dialog appears:

![Save As dialog](https://devnet.logianalytics.com/hc/article_attachments/4404920555671/svas.gif)

1. In the Save In section, browse to the folder where you want to save the analysis template in the server resource tree. The folder may be Public Reports or My Reports. You can use the button ![Up Level button](https://devnet.logianalytics.com/hc/article_attachments/4404926673687/btn_dshbrd_up.gif) to return to the parent folder.
   The root folder cannot be used to store resources into.

   The resource table shows the analysis templates in the current directory. Select the column names to change the order of the analysis templates in the table list if required.
2. In the File Name box, enter the name of the analysis template or use the default name, without suffix.
3. Select **OK** to save the analysis template. The saved template is linked to its original catalog.

To save a copy of an analysis template, select ![Save As button](https://devnet.logianalytics.com/hc/article_attachments/4404920358039/btn_saveas.gif) on the toolbar or select **![Menu button](https://devnet.logianalytics.com/hc/article_attachments/4404920661911/btn_va_menu.gif)** > **Save As** to show the Save As dialog, and then do as above. If you are saving to an existing file, a Confirm dialog will be displayed asking whether you want to replace the file or [save a new version into the file](https://devnet.logianalytics.com/hc/en-us/articles/1500009649002-Creating-Versions#Save).

After saving the analysis template into the server resource tree, you can browse to its directory on the Logi JReport Console > Resources page and run it directly.

**Note:** You will not be able to save resources to some locations if you do not have the Write permission.
