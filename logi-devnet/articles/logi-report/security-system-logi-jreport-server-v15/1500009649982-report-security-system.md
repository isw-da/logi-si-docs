# Report Security System

This topic demonstrates the control of access to reports and to different subsets of data by means of defining report security. In general, report security can be classified into three groups: business view security, cached report bursting, record level and column level. Administrators can also create dynamic security policies for catalogs published to the server using security files, which may contain security definitions of record-level security (RLS), column-level securty (CLS) and business view security. After applying a security policy, an end user will only see what he or she is allowed to see.

The following topics explain the report security at each level in detail:

* [Cached Report Bursting](https://devnet.logianalytics.com/hc/en-us/articles/1500009674901-Cached-Report-Bursting)
