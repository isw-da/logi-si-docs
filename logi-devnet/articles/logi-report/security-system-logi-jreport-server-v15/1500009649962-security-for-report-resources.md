# Security for Report Resources

In Logi JReport Server, there are two types of security mechanisms for managing access and visibility of report resources.

One is based on setting attributes of a report to control access and to control visibility of different subsets of data based on user identity.
This control is set during the design step, carried forward when published, and handled at runtime without any special administrative action.

The other is based on administrative action in Logi JReport Server to define the access rules for report resources in the runtime environment.
This involves registering Logi JReport Server users, and setting permissions on published Report resources.

The following topics discuss these two mechanisms, respectively:

* [Report Security System](https://devnet.logianalytics.com/hc/en-us/articles/1500009649982-Report-Security-System)
* [Server Security System](https://devnet.logianalytics.com/hc/en-us/articles/1500009650022-Server-Security-System)
