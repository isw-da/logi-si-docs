# Managing Your Servers Logi JReport Server v16

This topic discusses the management of resources, versions, tasks, server data, cached report data, in-memory cubes and triggers on Logi JReport Server. After going through this topic, you will be able to manage your Logi JReport Server more effectively.

The following topics provide more information on management:

* [Managing Resources](https://devnet.logianalytics.com/hc/en-us/articles/1500009717721-Managing-Resources)
* [Managing Versions](https://devnet.logianalytics.com/hc/en-us/articles/1500009717921-Managing-Versions)
* [Managing Tasks](https://devnet.logianalytics.com/hc/en-us/articles/1500009691382-Managing-Tasks)
* [Managing Server Data](https://devnet.logianalytics.com/hc/en-us/articles/1500009691342-Managing-Server-Data)
* [Managing Cached Report Data](https://devnet.logianalytics.com/hc/en-us/articles/1500009717701-Managing-Cached-Report-Data)
* [Managing In-Memory Cubes](https://devnet.logianalytics.com/hc/en-us/articles/1500009691322-Managing-In-memory-Cubes)
* [Managing Triggers](https://devnet.logianalytics.com/hc/en-us/articles/1500009691402-Managing-Triggers)

**Note:** A [Logi JReport Live license for Logi JReport Server](https://devnet.logianalytics.com/hc/en-us/articles/1500009691022-Logi-JReport-licenses#ServerLive) is required in order to use web report related features, and a [JDashboard license](https://devnet.logianalytics.com/hc/en-us/articles/1500009691022-Logi-JReport-licenses#JDashboard) for dashboard and library component related features. If you do not have the licenses, contact your Logi Analytics account manager to obtain them.
