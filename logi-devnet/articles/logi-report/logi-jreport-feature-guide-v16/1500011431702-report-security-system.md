# Report Security System

Report security is based on a type of filter control that limits the data shown on a report according to the user's access permissions. No matter to whom you need to provide information, Logi JReport Designer allows you to control access to reports and different subsets of data by means of defining report security.

Generally, report security can be classified into these groups: business view security, cached report bursting, record level and column level. After which, users will then only see what the access privileges allow. For more information about defining report security, see [Report Security](https://devnet.logianalytics.com/hc/en-us/articles/1500010063221-Defining-Report-Security).

After a catalog is published to Logi JReport Server, new report security policy can be applied to it dynamically which is called dynamic security.

This topic introduces the following security policies:

* [Business View Security](https://devnet.logianalytics.com/hc/en-us/articles/1500011431682-Business-View-Security)
* [Cached Report Bursting](https://devnet.logianalytics.com/hc/en-us/articles/1500011463701-Cached-Report-Bursting)
* [Record Level and Column Level Security](https://devnet.logianalytics.com/hc/en-us/articles/1500011431662-Record-Level-and-Column-Level-Security)
* [Dynamic Security](https://devnet.logianalytics.com/hc/en-us/articles/1500011463681-Dynamic-Security)
