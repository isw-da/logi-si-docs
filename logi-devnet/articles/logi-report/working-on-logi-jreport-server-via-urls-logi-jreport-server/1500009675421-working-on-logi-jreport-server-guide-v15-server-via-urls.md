# Working on Logi JReport Server via URLs Logi JReport Server Guide v15

Logi JReport Server provides URL commands for running reports and working with the server. You can directly type the whole URL in a web browser or create it in your web application to access the destination. The HTTP methods GET and POST are available for almost all of the commands. The length of attributes in the POST method is unlimited while that in the GET method is limited.

The URL examples in the following topics use the default context path in a standalone environment if with no special clarification. If the [default context path is changed](https://devnet.logianalytics.com/hc/en-us/articles/1500009644442-Changing-the-Server-Context-Path), you need to use the new context path in the URLs.

You can use URL to do the following:

* [Running Reports](https://devnet.logianalytics.com/hc/en-us/articles/1500009650462-Running-Reports)
* [Scheduling Reports](https://devnet.logianalytics.com/hc/en-us/articles/1500009650522-Scheduling-Reports)
* [Creating Reports](https://devnet.logianalytics.com/hc/en-us/articles/1500009650402-Creating-Reports)
* [Using the Authentication Properties](https://devnet.logianalytics.com/hc/en-us/articles/1500009650382-Using-the-Authentication-Properties-)
* [Working With Logi JReport Server](https://devnet.logianalytics.com/hc/en-us/articles/1500009650542-Working-with-Logi-JReport-Server-Guide-v15-Server)
