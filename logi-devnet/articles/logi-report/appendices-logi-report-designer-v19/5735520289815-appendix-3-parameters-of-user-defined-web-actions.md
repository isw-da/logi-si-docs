# Appendix 3: Parameters of User-Defined Web Actions

Except for Filter, Sort, Parameter, Property, and SendMessage, all the other web actions do not have a web action builder. You can apply these web actions directly or with some parameter values. This topic introduces the usage of these web actions and the parameters each web action requires.

| Web Action | Parameter | Description |
| --- | --- | --- |
| user\_atuoRefreshALLDataSet |  | Refreshes the report data automatically at certain interval. |
|  | time | Specifies the interval at which Page Report Studio refreshes the report data automatically. |
| user\_bottomN |  | Shows records with the field value equal to one of the last N field values in a data component. |
|  | instanceName | Specifies the instance name of the data component. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
|  | columnMappingName | Specifies the DBField name based on query. Value Format: MappingName |
|  | value | Specifies the value for N, which is a positive integer. |
| user\_changeComboChartType |  | Changes combo chart type. |
|  | instanceName | Specifies the instance name of the chart. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
|  | isPrimaryAxis | Specifies whether the axis displaying the type is the primary axis or not. |
|  | index | Specifies the index of the type, which is an integer equal to or larger than 0 and equal to or less than the value after the total number of all chart types minus 1. |
|  | type | Specifies the chart type to change the original type to. You can choose among these 2-D types: bar/bench, line, area, and stock. Valid values should use the chart type names listed right after the sample images in [Chart Types](https://devnet.logianalytics.com/hc/en-us/articles/5735527491351-Chart-Types). |
| user\_changeCompProperty |  | Changes a specific property of an object. |
|  | instanceName | Specifies the instance name of the object. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
|  | [propertyName](#Property) | Specifies the property name. |
|  | value | Specifies the property value. |
|  | isBy | Specifies whether the property value is a formula or real value. |
| user\_changeGenericChartType |  | Changes single chart type. |
|  | instanceName | Specifies the instance name of the chart. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
|  | isPrimaryAxis | Specifies whether the axis displaying the type is the primary axis or not. |
|  | type | Specifies the chart type to change the original type to. Valid values should use the chart type names listed right after the sample images in [Chart Types](https://devnet.logianalytics.com/hc/en-us/articles/5735527491351-Chart-Types). |
| user\_changeStyleTo |  | Changes the report style. |
|  | styleName | Specifies the name of a CSS or XSD style, for example, CSS style name: Classic, XSD style name: Classic\*. |
| user\_chartToCrosstab |  | Shows the To Crosstab dialog box for converting a chart to a crosstab. |
|  | instanceName | Specifies the instance name of the chart. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
| user\_closeReportSet |  | Closes the report. |
| user\_closeRpt |  | Closes the report tab. |
| user\_crosstabToChart |  | Shows the To Chart dialog box for converting a crosstab to a chart. |
|  | instanceName | Specifies the instance name of the crosstab. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
| user\_deleteRpt |  | Deletes the report. |
| user\_downloadReport |  | Downloads report in the specified format. Its usage is similar to user\_oneStepExport. |
|  | type | Represents the format: 0 for HTML, 2 for PDF, 3 for Postscript, 4 for RTF, 5 for TEXT, 6 for Excel, and 7 for XML. |
|  | [optionValues](#ExportOption) | Specifies the downloading options. Value Format: ['Export\_Option\_Key=Value','Export\_Option\_Key=Value',...] |
| user\_drillTo |  | Drills to a specific dimension on a data component. |
|  | instanceName | Specifies the instance name of the data component. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
|  | columnBLName | Specifies the qualified name of the group based on business view. Value Format: BusinessViewTreeRootNode.ParentNode.ViewElementName (There can be no parent node or 1 or more levels of parent nodes.) |
| user\_exit |  | Closes the report. |
| user\_filter |  | Filters a data component on a specified field. |
|  | instanceName | Specifies the instance name of the data component. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
|  | columnMappingName | Specifies the DBField name based on query. Value Format: MappingName |
|  | operator | Specifies the operator for the filter criterion. =: "eq"  >: "gt"  >=: "geq"  <: "lt"  <=: "leq"  !=: "neq"  IN: "in"  NOT IN: "notin"  LIKE: "like"  NOT LIKE: "notlike"  BETWEEN:"between"  NOT BETWEEN:"notbetween"  IS NULL:"isnull"  IS NOT NULL:"isnotnull" |
|  | value | Specifies the operand. |
|  | logic | Specifies the logic relationship between two neighboring filter criteria. The logic can be "AND" or "OR". |
|  | valueFormat | Specifies the format for the value. Usually, you only need to specify this parameter for fields of Date/Time type. If you do not specify the parameter, Page Report Studio accepts the default format for the value. Data type: String |
| user\_firstPage |  | Turns to the first page. |
| user\_groupFlex |  | Expands or collapses a specific group. |
|  | groupControlObj | Specifies the group to be expanded or collapsed. Data type: HTML object. You can use "this" to represent the current group. |
| user\_lastPage |  | Turns to the last page. |
| user\_nextPage |  | Turns to the Next Topic. |
| user\_oneStepExport |  | Exports report to a specified format. |
|  | type | Represents the format: 0 for HTML, 2 for PDF, 3 for Postscript, 4 for RTF, 5 for TEXT, 6 for Excel, and 7 for XML. |
|  | [optionValues](#ExportOption) | Specifies the export options and their values. Value Format: ['Export\_Option\_Key=Value','Export\_Option\_Key=Value',...] |
| user\_oneStepFilter |  | Filters a data component on multiple fields. |
|  | columnMappingNames | Specifies the DBField names based on query. Value Format: [MappingName1,MappingName2,MappingName3,...] |
|  | operators | Specifies the operators for the filter criteria. Available values are the same as those for operator of [user\_filter](#Operator). |
|  | values | Specifies the operands. |
|  | logics | Specifies the logic relationships between two neighboring filter criterions. The logic can be "AND" or "OR". |
|  | instanceName | Specifies the instance name of the data component. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
| user\_oneStepSearch |  | Locates a string in the report. |
|  | value | Specifies the string to find. |
|  | isContent | Specifies whether to search the whole report content or a specified field. |
|  | columnMappingName | Specifies the DBField name based on query. Value Format: MappingName |
|  | isUp | Specifies whether to search from the bottom to the top. |
|  | isMatchCase | Specifies whether to match case when searching. |
|  | isWordOnly | Specifies whether to match whole word. |
| user\_oneStepSort |  | Sorts a data component on multiple fields. |
|  | columnMappingNames | Specifies the DBField names based on query. Value Format: [MappingName1,MappingName2,MappingName3,...] |
|  | orders | Specifies the sorting orders. Value Format: [true,false,false,...] ("true" for ascending, "false" for descending). |
|  | instanceName | Specifies the instance name of the data component. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
| user\_openNewReport |  | Opens a report. |
|  | catalog | Specifies the catalog of the report. You should use the catalog file path in the resource tree of Server. /USERFOLDERFATH/admin/ stands for the My Reports folder, and / stands for the Public Reports folder. |
|  | rptName | Specifies the report name. You should use the report file path in the resource tree of Server. /USERFOLDERFATH/admin/ stands for the My Reports folder, and / stands for the Public Reports folder. |
|  | newWindow | Specifies whether to open a new browser window for the report. |
|  | params | Specifies parameter names and values for opening the report. Value Format: [[[name1],[value1]],[[name2],[value2]],...] |
| user\_prevPage |  | Turns to the Previous Topic. |
| user\_popCustomizedPage() |  | Pops up the Customized Page dialog box, which by default displays all the parameters used in the current report if it contains parameters. |
| user\_redo |  | Redoes an action. |
| user\_refresh |  | Refreshes the report. |
| user\_removeFilter |  | Removes the filters on the specified field or all the fields in a data component. |
|  | instanceName | Specifies the instance name of the data component. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
|  | columnMappingName | Specifies the DBField name based on query. If you specify the parameter, Page Report Studio removes the filters on this field in the component; if you do not it, Page Report Studio removes filters on all the fields of the component. Value Format: MappingName |
| user\_renameRpt |  | Shows the Rename Report dialog box. |
| user\_reset |  | Resets the report. |
|  | instanceName | Specifies the instance name of the report. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
| user\_rotateCrosstab |  | Rotates a crosstab. |
|  | instanceName | Specifies the instance name of the crosstab. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
| user\_saveRpt |  | Saves the report. |
| user\_showDHTMLView |  | Shows the Resource View panel. |
| user\_showFilterDialog |  | Shows the Filter dialog box. |
| user\_showHelp |  | Shows the help about a specific dialog box or shows the index page of the Page Report Studio help documentation if null. |
|  | helpId | Specifies the ID of the specific Page Report Studio dialog box help topic, which is a five-digit integer starting with 4, for example 40034. You can refer to the file Hyperlink.txt in the Designer installation root to check for the help files that are linked with IDs starting with 4. |
| user\_showNewRptDialog |  | Shows the New Report dialog box. |
| user\_showOpenRptDialog |  | Shows the Open Report dialog box. |
| user\_showPageSetupDialog |  | Shows the Page Properties dialog box. |
| user\_showPrintDialog |  | Shows the Printable Version dialog box. |
| user\_showSaveAsDialog |  | Shows the Save As dialog box. |
| user\_showSaveResultDialog |  | Shows the Export dialog box. |
| user\_showSearchDialog |  | Shows the Search dialog box. |
| user\_showSortDialog |  | Shows the Sort dialog box. |
| user\_showTOC |  | Shows the TOC Browser. |
| user\_showToolbox |  | Shows the Toolbox. |
| user\_showUserPanel |  | Shows the User Information bar. |
| user\_sort |  | Sorts a data component on a specific field. |
|  | instanceName | Specifies the instance name of the data component. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
|  | columnMappingName | Specifies the DBField name based on query. Value Format: MappingName |
|  | order | Specifies the sorting order: "true" for ascending and "false" for descending. |
| user\_stopAutoRefreshDataSet |  | Stops Page Report Studio from refreshing the report data automatically. |
| user\_topN |  | Shows records with the field value equal to one of the top N field values in a data component. |
|  | instanceName | Specifies the instance name of the data component. Since Logi Report does not expose instance name to users, you can use the display name as the value here. However, the value should follow certain format. Value Format: &DisplayName |
|  | columnMappingName | Specifies the DBField name based on query. Value Format: MappingName |
|  | value | Specifies the value for N, which is a positive integer. |
| user\_undo |  | Undoes an action. |
| user\_zoomTo |  | Zooms the report in or out to a specific magnification. |
|  | zoomSize | The magnification ratio in percent. It should be an integer between 0 and 400. |

![Note icon](https://devnet.logianalytics.com/hc/article_attachments/9898403122327/noteicon.jpg) If you are very familiar with the usages of these web actions, you can select the **Input an Action** option, and then specify the action name with parameter.
