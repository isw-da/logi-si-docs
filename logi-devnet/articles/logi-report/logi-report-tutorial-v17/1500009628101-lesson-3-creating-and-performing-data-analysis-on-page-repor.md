# Lesson 3: Creating and Performing Data Analysis on Page Reports

Using Page Report Wizard you can create a single component page report which is viewed in Page Report Studio. Page Report Studio provides the user a page mode view of the report very similar to a printed report.

There are several types of reports in the Page Report Wizard which collect information for you: blank (no component), banded, crosstab, table, and chart. You can add more components to a report after it is generated by the wizard. In this lesson, we will create a page report that contains three report tabs, and then perform data analysis on them one by one.

This lesson contains the following tasks:

## Task 1: Create a Banded Report

1. In the [Logi Report Server Start Page](https://devnet.logianalytics.com/hc/en-us/articles/1500009627641-Lesson-1-Starting-Logi-Report-Server), select **Page Report** in the Create category.
2. In the displayed page, select the **SampleReports** folder and **SampleReports.cat** catalog file, then select **OK**.

   ![Select Catalog](https://devnet.logianalytics.com/hc/article_attachments/4404904499863/adhc_lsn3_tsk1_cat.gif)
3. In the New Page Report dialog box, type **Order Details** in the Report Title text field, select **Banded** in the layout box, then select **OK**.

   ![Select Banded for Page Report](https://devnet.logianalytics.com/hc/article_attachments/4404916847895/adhc_lsn3_tsk1_nwrptst.gif)

   The Banded Wizard appears.
4. In the Data screen, select **WorldWideSalesBV** in Data Source 1 from the Available Data Resources box.
   Select **Next**.
5. In the Display screen, expand the **Products** category in the Resources box, add **Product Name** as a detal field to the right box, then expand the **Orders Detail** category and add **Quantity**, **Unit Price**, and **Total** one by one to the right box. Select **Next**.

   ![Add Banded Display Fields](https://devnet.logianalytics.com/hc/article_attachments/4404904500119/adhc_lsn3_tsk1_bddsply.gif)
6. In the Group screen, add **Country** in the Customers category as the group-by field, keep Ascend as the sort manner, and then select **Next**.
7. In the Summary screen, select the **Country** group in the right box, then add **Total Sales** to summarize the total sales for each country. Select **Next**.

   ![Add Banded Summary Field](https://devnet.logianalytics.com/hc/article_attachments/4404916848151/adhc_lsn3_tsk1_bdsum.gif)
8. Skip the Query Filter screen and select **Next**.
9. In the Style screen, set the style to **Classic**, then select **Finish** to create the report.

   The report displays as follows in Page Report Studio:

   ![Banded Report](https://devnet.logianalytics.com/hc/article_attachments/4404904500375/adhc_lsn3_tsk1_bdrpt.gif)
10. Select **Menu** > **File** > **Rename Report Tab**.
11. In the Rename Report Tab dialog box, type **OrderDetails** as the report tab name, then select **OK**.
12. Select the **Save** button ![Save button](https://devnet.logianalytics.com/hc/article_attachments/4404904500759/btn_save.gif) on the toolbar.
13. In the Save As dialog box, type **CustomerOrders.cls** in the File Name text field, then select **OK** to save the report into the Public Reports > SampleReports folder in the server resource tree.

## Task 2: Create a Table Report

1. In the current open report window, select the **New Report Tab**  button ![New Report Tab button](https://devnet.logianalytics.com/hc/article_attachments/4404916848407/btn_nwrpt.gif) on the toolbar.
2. In the New Report Tab dialog box, type **Customer Information** in the Report Title text field, select **Table (Group Above)**  in the layout box, then select **OK**. The Table Wizard appears.
3. In the Data screen of the wizard, select **CoffeeSalesUSA** in Data Source 1, then select **Next**.
4. In the Display screen, add **Customer Name**, **Phone**, **Postal Code** , and **Address 1** one by one as the detail fields in the table. Select **Next**.

   ![Add Table Display Fields](https://devnet.logianalytics.com/hc/article_attachments/4404904501143/adhc_lsn3_tsk2_tbldsply.gif)
5. Skip the Group, Summary and Query Filter screens.

1. In the Style screen, set the style to **Classic**, then select **Finish** to create the table report.

   The table report appears as follows:

   ![Table Report](https://devnet.logianalytics.com/hc/article_attachments/4404904501527/adhc_lsn3_tsk2_tblrpt.gif)
2. Select **Menu** > **File** > **Rename Report Tab** to rename the report tab as **CustomerInfo**.
3. Select the **Save** button ![Save button](https://devnet.logianalytics.com/hc/article_attachments/4404904500759/btn_save.gif) on the toolbar. In the Save Report Template dialog box, select **Yes** to save the report.

## Task 3: Create a Crosstab Report

1. In the current open report window, select the **New Report Tab** button ![New Report Tab button](https://devnet.logianalytics.com/hc/article_attachments/4404916848407/btn_nwrpt.gif) on the toolbar.
2. In the New Report Tab dialog box, type **Product Sales** in the Report Title text field, select **Crosstab** in the layout box, then select **OK**. The Crosstab Wizard appears.
3. In the Data screen, select **WorldWideSalesBV** in Data Source 1 and then select **Next**.
4. In the Display screen, add **Country** from the Customers category in the Resources box as the row field and **Category** from the Products category as the column field (keep Ascend as the sort manner for both columns and rows), then add **Total Sales** from the Orders Detail category as the summary field. Select **Next**.

   ![Add Crosstab Display Fields](https://devnet.logianalytics.com/hc/article_attachments/4404916848663/adhc_lsn3_tsk3_crstbdsply.gif)
5. Skip the Query Filter screen. Select **Next**.
6. In the Style screen, set the crosstab style as **Classic**, then select **Finish** to create the crosstab report.

   The crosstab report appears as follows:

   ![Crosstab Report](https://devnet.logianalytics.com/hc/article_attachments/4404904502039/adhc_lsn3_tsk3_crstbrpt.gif)
7. Select **Menu** > **File** > **Rename Report Tab** to rename the report tab as **ProductSales**.
8. Select ![Save button](https://devnet.logianalytics.com/hc/article_attachments/4404904500759/btn_save.gif) on the toolbar. In the Save Report Template dialog box, select **Yes** to save the report.

## Task 4: Insert a Chart

In this task, we will insert a chart below the crosstab created in Task 3.

1. Drag **Chart** from the Toolbox panel to place it under the last row of the crosstab.

   ![Insert Chart Below Crosstab](https://devnet.logianalytics.com/hc/article_attachments/4404904502295/adhc_lsn3_tsk4_drg.gif)

   The Chart Wizard appears.
2. In the Data screen, select **WorldWideSalesBV** in Data Source 1, then select **Next**.
3. In the Type screen, keep the default chart type Clustered Bar 2-D and select **Next**.
4. In the Display screen, add **Sales Quarter**  from the Orders Detail category to the Category box and **Product Type** in the Products category to the Series box, then add **Total Sales** in the Orders Detail category to the Show Values box.

   ![Add Chart Display Fields](https://devnet.logianalytics.com/hc/article_attachments/4404904502807/adhc_lsn3_tsk4_chtdsply.gif)
5. Select **Finish** to create the chart. The chart appears as follows:

   ![Chart Component](https://devnet.logianalytics.com/hc/article_attachments/4404904503063/adhc_lsn3_tsk4_chtrpt.gif)
6. Select ![Save button](https://devnet.logianalytics.com/hc/article_attachments/4404904500759/btn_save.gif) on the toolbar. In the Save Report Template dialog box, select **Yes** to save the report.

## Task 5: Analyze Data of the Banded Object

1. In the current report window, select **OrderDetails** from the Go To drop-down list on the toolbar to switch to this report tab.

   ![Switch to Banded Report Tab](https://devnet.logianalytics.com/hc/article_attachments/4404916848919/adhc_lsn3_tsk5_rptstbr.gif)

We want to make the orders in each country further grouped by order ID, so first we need to add another group into the banded object.

2. Close the Toolbox panel on the left.
3. Expand the **Orders Detail**  category in the Resource View panel, drag **Order ID**  below the Australia group header and drop it when a blue line appears and a tip Group Header shows up.

   ![Add Banded Group](https://devnet.logianalytics.com/hc/article_attachments/4404904503319/adhc_lsn3_tsk5_addgrp.gif)

Next we want to add a column about the discount of each product.

1. Drag **Discount** from the Resource View panel to the detail panel of the banded object:

   ![Add Banded Detail](https://devnet.logianalytics.com/hc/article_attachments/4404904503575/adhc_lsn3_tsk5_adddtl.gif)

   The new group and detail column display in the banded object as follows:

   ![Banded with Newly Added Detail](https://devnet.logianalytics.com/hc/article_attachments/4404916849175/adhc_lsn3_tsk5_discnt.gif)

Now let's do some adjustments to improve the appearance of the banded object.

1. Right-click the **Discount** label in the banded header and select **Properties** from the shortcut menu. The Label Properties dialog box appears.
2. In the General tab, edit the Width property to **1.04**, then select the **Font** tab and change Horizontal Alignment to **right**. Select **OK** to apply the property settings.

   The Discount label is now aligned with the values in the same column.

   ![Edit Label Properties](https://devnet.logianalytics.com/hc/article_attachments/4404904503959/adhc_lsn3_tsk5_prpty.gif)

There are two group levels in the banded object, to easily distinguish them we will change the background color of the Order ID group.

1. Right-click any of the Order ID group header panel, the panel of 3033 for example, then select **Properties** on the shortcut menu.
2. In the General tab of the Banded Panel Properties dialog box, set the Background property to **#668cb2**, then select **OK** to confirm.

There is a blank panel before each new order ID begins. This is the group footer panel of the Order ID group. Since we will not add any data to the panel, next we will hide it.

1. Right-click any of the Order ID group footer panel, the blank footer of 3033 for example, then select **Hide** from the shortcut menu.

   ![Hide Category Group Footer](https://devnet.logianalytics.com/hc/article_attachments/4404904504215/adhc_lsn3_tsk5_grpftr.gif)

   After making the appearance adjustments the banded object looks as follows:

   ![Banded Without Category Group Footer](https://devnet.logianalytics.com/hc/article_attachments/4404916849431/adhc_lsn3_tsk5_frmt.gif)

Next, we will sort and filter the banded object. The totals will be sorted in descending order for each group. The records will be filtered by discount to show orders in which the discount is 10 point only.

1. Select the **Sort** button ![Sort button](https://devnet.logianalytics.com/hc/article_attachments/4404904504599/btn_sort.gif) on the toolbar to open the Sort dialog box.
2. In the Sort dialog box, choose **Total** from the field drop-down list, select the sorting manner as **Descend**, and then select **OK** to apply the settings.

   ![Sort Total Ascend](https://devnet.logianalytics.com/hc/article_attachments/4404916849943/adhc_lsn3_tsk5_srt.gif)
3. Right-click any value in the Discount column, then on the shortcut menu, select **Filter** > **10.00**.

   ![Filter Discount](https://devnet.logianalytics.com/hc/article_attachments/4404904504855/adhc_lsn3_tsk5_fltr.gif)

   The banded object fianlly turns out as follows:

   ![Banded After Sort and Filter](https://devnet.logianalytics.com/hc/article_attachments/4404904505111/adhc_lsn3_tsk5_final.gif)
4. Select ![Save button](https://devnet.logianalytics.com/hc/article_attachments/4404904500759/btn_save.gif) on the toolbar. In the Save Report Template dialog box, select **Yes** to save the report.

In the above steps, we used two ways to sort and filter the report results: one by dialog box and the other by right-click menu command. In fact, both operations can be achieved by either way.

## Task 7: Analyze Data of the Crosstab

In this task, we will demonstrate the Drill feature of Page Report Studio based on the crosstab in the ProductSales report tab. Logi Report supports four types of drill actions: drill-up, drill-down, drill-to, and drill-to-by-value. They can be performed on crosstabs, charts, grouped tables, and banded objects. However to perform the drill-up and drill-down actions on a data component, the business view used by the component should contain predefined hierarchies, so that you are able to drill from one group level up or down to another within a hierarchy.

1. Select **ProductSales** from the Go To drop-down list on the toolbar to switch to the report tab, which contains a crosstab and a chart.

Since Country is a middle level predefined in the hierarchy Geography of WorldWideSalesBV, we can drill it up or down to the adjacent level.

1. Right-click any country in the row header, then on the shortcut menu select **Drill Up** > **Region**.

   ![Drill up to Region](https://devnet.logianalytics.com/hc/article_attachments/4404904507799/adhc_lsn3_tsk7_drlup.gif)

   The crosstab refreshes to show as follows:

   ![Drill up to Region Result](https://devnet.logianalytics.com/hc/article_attachments/4404916850583/adhc_lsn3_tsk7_drluprst.gif)

Next we want to drill down from Region to Country, however it is not simply a reversed process of drilling up from Country to Region. When you drill down to a lower group level, Logi Report will apply the currently selected value as a filter condition to show its data in the lower group level only. Therefore after drill-down you cannot use drill-up to go back to the previous state, instead you can simply perform an Undo action.

1. Right-click **Asia-Pacific** in the row header and select **Drill Down** > **Country** from the shortcut menu.
   Now only countries in Asia-Pacific is displayed in the row header:

   ![Drill down to Country](https://devnet.logianalytics.com/hc/article_attachments/4404916850839/adhc_lsn3_tsk7_drldw.gif)

We can further drill Country down to State and then to City in this way. Here let's leave the crosstab as it is and try the other two drill actions.

1. Right-click any product category in the column header, then on the shortcut menu select **Drill To** > **Sales Year**.

   ![Drill To Menu List](https://devnet.logianalytics.com/hc/article_attachments/4404916851095/adhc_lsn3_tsk7_drlto.gif)

   The crosstab is regenerated to show sales in the Asia-Pacific countries in 2015 and 2016:

   ![Drill to Sales Year](https://devnet.logianalytics.com/hc/article_attachments/4404904508311/adhc_lsn3_tsk7_drltorst.gif)
2. Right-click **2016** in the column header and select **Drill to by Value** > **Product Name** from the shortcut menu. The crosstab now shows each product's sales in the Asia-Pacific countries in 2016.

   ![Drill to by Value](https://devnet.logianalytics.com/hc/article_attachments/4404904508567/adhc_lsn3_tsk7_drltobyvl.gif)

Lastly we want to rotate the crosstab, that is to switch its column and row headers so that the crosstab will not be split into two.

1. Select any aggregation cell in the crosstab to select the crosstab, then select the **Rotate** button ![Rotate button](https://devnet.logianalytics.com/hc/article_attachments/4404904508823/btn_rotate.gif) on the toolbar.
2. Resize the crosstab row header by dragging its right boundary to the right to make values in the row header completely displayed.

   The crosstab finally changes to:

   ![Crosstab Final Result](https://devnet.logianalytics.com/hc/article_attachments/4404916851351/adhc_lsn3_tsk7_final.gif)
3. Select ![Save button](https://devnet.logianalytics.com/hc/article_attachments/4404904500759/btn_save.gif) on the toolbar. In the Save Report Template dialog box, select **Yes** to save the report.

## Task 8: Analyze Data of the Chart

Now let's focus on the chart below the crosstab in the ProductSales report tab. In Logi Report, a chart can be converted to a crosstab, and vice versa. We will convert the chart to a crosstab first.

1. Right-click on the chart and select **To Crosstab** from the shortcut menu. The To Crosstab dialog box appears.
2. The Resources box in the Display tab lists the fields used in the crosstab. Add **Product Type** to the Columns box, **Sales Quarter** to the Rows box, and **Total Sales** to the Summaries box. Select **OK** to generate the crosstab.

   ![Convert Chart to Crosstab](https://devnet.logianalytics.com/hc/article_attachments/4404904509335/adhc_lsn3_tsk8_dsply.gif)

   The chart is converted to a crosstab as follows:

   ![Converted Crosstab](https://devnet.logianalytics.com/hc/article_attachments/4404904509591/adhc_lsn3_tsk8_crstb.gif)

Next we want to return the crosstab to chart to go on analyzing with chart. We want to change the chart type to line chart.

1. Select the **Undo** button ![Undo button](https://devnet.logianalytics.com/hc/article_attachments/4404916851607/btn_undo.gif) on the toolbar to cancel the operation of converting chart to crosstab.
2. Right-click on the chart, and on the shortcut menu, select **Chart Type** > **Line** > **Line 2-D**. The chart appears as follows:

   ![Change Chart Type](https://devnet.logianalytics.com/hc/article_attachments/4404904509847/adhc_lsn3_tsk8_line.gif)

In Page Report Studio, the definition of a chart can be changed at any time. Next we will use different type and fields to re-define the chart.

1. Right-click anywhere in the chart except the legend and select **Chart Wizard** on the shortcut menu**.** The Chart Definition dialog box appears.
2. In the Type tab, select the  **Bench** chart type, keep the default sub type as Clustered Bench 2-D.
3. Select the **Display** tab, remove the fields in the Category and Series boxes respectively, then expand the **Products** category in the Resources and add **Category**to the Category box, add **RegionAbbr** in Customers to the Series box, and **Total Quantity**  in Orders Detail to the Show Values box. Select **OK** to create the chart.

   ![Define the Bench Chart](https://devnet.logianalytics.com/hc/article_attachments/4404904510103/adhc_lsn3_tsk8_chtdfn.gif)

   The new bench chart displays as follows:

   ![Bench Chart](https://devnet.logianalytics.com/hc/article_attachments/4404904510359/adhc_lsn3_tsk8_bench.gif)

Properties of the elements in a chart, such as the platform, paper, legend and axes, can be customized using corresponding format dialog boxes to suit your requirements. Next we will edit some properties of the tick marks on the Y axis to briefly introduce the Chart Format feature.

1. Right-click anywhere in the chart except the legend and select **Format Axis** > **Format Value (Y) Axis**from the shortcut menu.
2. In the Format Value (Y) Axis dialog box, switch to the **Tick Mark** tab, in the Major Tick Mark sub tab, select **Cross**, clear **Correlate with Axis** and type **#ff3333** in the Color text box, specify to show data labels every **2** tick marks. Select **OK** to apply the properties.

   ![Format Value Axis](https://devnet.logianalytics.com/hc/article_attachments/4404904510615/adhc_lsn3_tsk8_yfmt.gif)

   The tick marks and data labels on the Y axis apply the new properties and display as follows:

   ![Final Chart](https://devnet.logianalytics.com/hc/article_attachments/4404916852119/adhc_lsn3_tsk8_final.gif)
3. Select ![Save button](https://devnet.logianalytics.com/hc/article_attachments/4404904500759/btn_save.gif) on the toolbar. In the Save Report Template dialog box, select **Yes** to save the report.
