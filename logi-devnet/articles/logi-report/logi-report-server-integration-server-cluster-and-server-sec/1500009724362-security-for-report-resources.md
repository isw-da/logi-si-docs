# Security for Report Resources

In Logi Report Server, there are two types of security mechanisms for managing access and visibility of report resources. One is based on setting attributes of a report to control access and to control visibility of different subsets of data based on user identity.
This control is set during the design step, carried forward when published, and handled at runtime without any special administrative action. The other is based on administrative operations in Logi Report Server to define the access rules for report resources in the runtime environment.
This involves registering users, groups, and roles in the server security system, and assigning them permissions on report resources.

The following topics provide more information on these two mechanisms:

* [Report Security System](https://devnet.logianalytics.com/hc/en-us/articles/1500009751181-Report-Security-System)
* [Server Security System](https://devnet.logianalytics.com/hc/en-us/articles/1500009724482-Server-Security-System)
