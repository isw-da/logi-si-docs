# Working with APIs Logi JReport Server v16

APIs provide a way to build Logi JReport components and services into other applications, thus integrating Logi JReport with existing platform infrastructure. This topic is aimed at Java developers wanting to understand how Logi JReport works, and how to write Java code that uses the APIs effectively.

The comprehensive Java doc matrix covering all the classes and methods of the APIs is available in `<install_root>\help\api`. This topic is an introduction and overview of the APIs so that the vastness of the Java docs will be easier to master.

Beyond that are explanations of use cases, with examples of code.

Logi JReport has the following sets of APIs:

* [Server API](https://devnet.logianalytics.com/hc/en-us/articles/1500009686162-Server-API)
* [Catalog API](https://devnet.logianalytics.com/hc/en-us/articles/1500009711841-Catalog-API)
* [Design API](https://devnet.logianalytics.com/hc/en-us/articles/1500009711861-Design-API-)
* [Exit Functions](https://devnet.logianalytics.com/hc/en-us/articles/1500009686142-Exit-Functions)
* [Security Context Support](https://devnet.logianalytics.com/hc/en-us/articles/1500009711881-Security-Context-Support)
