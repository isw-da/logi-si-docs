# Encoding Dialog Box Properties

You can use the Encoding dialog box to specify the encoding for the detail report. This topic describes the options in the dialog box.

When you link a trigger field of a master report to a detail report in Logi Report Designer, if you did not select the option Use the same encoding and DB settings for the detail report as that of the mater report, after you publish the report to Logi Report Server, run it in Page Report Studio, and select the trigger field, Server displays the Encoding dialog box.

![Encoding dialog](https://devnet.logianalytics.com/hc/article_attachments/4404880541335/encode.gif)

**Page Report**

Displays the path and name of the current page report.

**Catalog**

Displays the path and name of the current catalog.

**Enable Converting Encoding**

Specifies to enable the conversion of encoding. From the Before Converting and After Converting drop-down lists, select the encoding.

**Encoding**

Specifies the encoding for the detail report.

**Connect to [Data Source Name]**

Specifies the DB user and password with which you want to connect to the data source that the detail report uses.

* **Use the DB user and password defined in catalog**  
   Select this option to use the DB user and password defined in the catalog.
* **Use the DB User**  
   Select this option to specify another DB user and password instead of using the one defined in the catalog.

**OK**

Closes this dialog box and applies the settings to run the detail report.

**Cancel**

Closes this dialog box and runs the detail report without regard to the settings.

**Help**

Displays the help document about this feature.
