# Changing Resource Properties

You can edit the properties of any resource (including folder in the server resource tree), such as its node name, description, archive policy, user permissions and so on. To do this:

* For administrators, on the Logi JReport Administration > Resources page, browse to the row that the resource is in and select the **Properties** button ![](https://devnet.logianalytics.com/hc/article_attachments/4404926692119/btn_srv_prpty.gif) in the Control column. In the corresponding properties dialog, change the settings as required.
* End users can edit the properties of one or more resources at a time on the Logi JReport Console > Resources page.

  **To edit the properties of a single resource:**

  1. Browse to the folder where the resource is located.
  2. Select the resource row and then do any of the following:
     + Put the mouse pointer over the resource row and select the **Properties** button ![](https://devnet.logianalytics.com/hc/article_attachments/4404926692119/btn_srv_prpty.gif) on its floating toolbar.
     + Right-select in the resource row and select **Properties** from the shortcut menu.
     + Select **Tools** > **Properties** on the task bar.
  3. In the displayed properties dialog, change the property settings and [specify the permissions](https://devnet.logianalytics.com/hc/en-us/articles/1500009674981-Managing-Permissions) of different principals on the resource if available.

  **To edit the properties of multiple resources at one time:**

  1. Browse to the folder where the resources are located.
  2. Check the checkboxes ahead of the required resources. To select all the resources, check the top checkbox.
  3. Select **Tools** > **Properties** on the task bar or right-click in any selected resource row and select **Properties** from the shortcut menu. The [Multiple Type Properties](https://devnet.logianalytics.com/hc/en-us/articles/1500009671041-Multiple-Type-Properties) dialog is displayed.
  4. In the General tab, edit the description, status, linked catalog and version archive policy for the resources. Settings for description and archive policy apply to all the selected resources, while the status setting works on report resources and linked catalog on report, library component and folder resources only.
  5. If the resources are in the Public Reports or Public Components folder, the Permission tab is available. Specify the permissions of different principals on the resources.
  6. Select **OK** to accept the changes.
