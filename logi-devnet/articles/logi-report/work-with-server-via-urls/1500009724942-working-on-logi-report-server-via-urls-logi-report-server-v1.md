# Work WithLogi Report Server via URLs

Logi Report Server provides URL commands for running reports and working with the server. You can directly type the whole URL in a web browser or create it in your web application to access the destination. The HTTP methods GET and POST are available for almost all of the commands. The length of attributes in the POST method is unlimited while that in the GET method is limited.

The URL examples in this topic use the default context path in a standalone environment if with no special clarification. If the [default context path is changed](https://devnet.logianalytics.com/hc/en-us/articles/1500009718802-Changing-the-Server-Context-Path), you need to use the new context path in the URLs.

* [Using the Authentication Properties](https://devnet.logianalytics.com/hc/en-us/articles/1500009751661-Using-the-Authentication-Properties-)
* [Working with Logi Report Server](https://devnet.logianalytics.com/hc/en-us/articles/1500009725002-Working-with-Logi-Report-Server)
