# Working on Logi Report Server via URL Logi Report Server v18

Logi Report Server provides URL commands for running reports and working with the server. This topic describes how you can run, schedule, and create reports, and work with the server, dashboards, and Visual Analysis, via URL.

You can directly type the whole URL in a web browser or create it in your web application to access the destination. You can use the HTTP methods GET and POST for almost all the commands. Server does not limit the length of attributes in the POST method while it does in the GET method.

The URL examples in this topic use the default context path in a standalone environment if with no special clarification. If you changed the [default context path](https://devnet.logianalytics.com/hc/en-us/articles/4405683582231-Changing-the-Server-Context-Path), you need to use the new context path in the URL.
