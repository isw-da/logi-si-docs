# Add User Dialog

The Add User dialog appears when you select ![Add button](https://devnet.logianalytics.com/hc/article_attachments/4404893791255/btn_add.gif) and then select Add User from the drop-down menu in the [Security](https://devnet.logianalytics.com/hc/en-us/articles/1500009588521-Security-Dialog) dialog, [Edit Business View Security](https://devnet.logianalytics.com/hc/en-us/articles/1500009565182-Edit-Business-View-Security-Dialog) dialog or in the Security tab of the following dialogs: [Category Property](https://devnet.logianalytics.com/hc/en-us/articles/1500009564982-Category-Property-Dialog), [New View Element](https://devnet.logianalytics.com/hc/en-us/articles/1500009588021-New-View-Element-Dialog) or [Edit View Element](https://devnet.logianalytics.com/hc/en-us/articles/1500009586641-Edit-View-Element-Dialog).

It helps you to add a user to the security policy so as to define his permission on the related resource, and consists of the following tabs:

* [General](#General)
* [Relation](#Relation)

**OK**

Accepts all changes and closes the dialog.

**Cancel**

Does not retain any changes and closes the dialog.

**Help**

Displays the help document about this feature.

## General

Specifies the user information. [See the dialog](javascript:%20void(null)).

![Add User dialog - General](https://devnet.logianalytics.com/hc/article_attachments/4404893957399/adduser_gen.gif)

**User**

Specifies the name of the user.

**Source**

Displays the source of the user. It is read only.

**E-mail**

Specifies the e-mail address of the user that is to be added.

**Description**

Specifies the description of the user.

## Relation

Specifies the relationship of the user and the existing roles and groups. [See the dialog](javascript:%20void(null)).

![Add User dialog - Relation](https://devnet.logianalytics.com/hc/article_attachments/4404893957655/adduser_rel.gif)

**Roles**

All available roles will be listed here. If a role is checked, the user will belong to the role.

**Groups**

All available groups will be listed here. If a group is checked, the user will belong to the group.
