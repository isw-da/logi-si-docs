# UDF Classes Dialog

The UDF Classes dialog appears when you select Menu > File > Import UDF Classes in the [Formula Editor](https://devnet.logianalytics.com/hc/en-us/articles/1500009586941-Formula-Editor-Dialog). It helps you to import [user defined formula classes](https://devnet.logianalytics.com/hc/en-us/articles/1500009589721-User-Defined-Formula-Functions). [See the dialog](javascript:%20void(null)).

![UDF Classes dialog](https://devnet.logianalytics.com/hc/article_attachments/4404889314199/udfcls.gif)

The following are details about options in this dialog:

**User Defined Formula Classes**

Lists the imported user defined formula classes with package path and alias.

By default, each imported UDF class is given an alias which takes the name of the UDF class even though the alias is not shown. You can modify the alias by double-clicking the UDF class and then specify a new alias in the Update UDF Class Alias dialog. The new alias which is different from the UDF class name will be displayed right next to the UDF class and be bracketed by "[]". The format is like this: UDF\_Class\_Name[Alias\_Name].

![Add button](https://devnet.logianalytics.com/hc/article_attachments/4404893791255/btn_add.gif)

Opens the Import UDF Class dialog for importing a new UDF class. To import a UDF class, you need first add it to the class path of Logi JReport Designer and then provide the class name with full package path in the Import UDF Class dialog.

![Remove button](https://devnet.logianalytics.com/hc/article_attachments/4404889276311/btn_rmv.gif)

Removes the selected UDF class.

**OK**

Applies the changes and closes the dialog.

**Cancel**

Does not retain any changes and closes the dialog.

**Help**

Displays the help document about this feature.
