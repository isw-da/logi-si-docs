# Scheduling Tasks to Run Reports

By setting up schedule tasks on reports, you can make reports run automatically at one or more designated dates and times, and then publish the report results to different destinations in various formats such as PDF, HTML, and Excel. This topic describes how you can schedule to run one or multiple reports on the Server Console.

Scheduled reports can greatly improve the performance of the reporting service in an organization, for example, reports can run during low network traffic times. Scheduled reports can also capture a period exactly, for example the last day of the quarter.

Logi Report Server supports delivering report results to the following destinations: To Version, To Disk, To E-mail, To Printer, To Fax, and To FTP. You can create a schedule task on one report to publish its result to all of the destinations at a time, however, when you schedule a task based on multiple reports, Server only supports To Version and To E-mail.
