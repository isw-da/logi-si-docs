# Using a Formula (e.g., Custom Aggregation) to Get a Result of Current Quarter/Month Compared to Last Year

Custom aggregations are special aggregations used for sophisticated mathematic and logic calculation purposes. They are created using the Formula Editor and are defined based on business view elements and built-in functions including aggregation functions, mathematic and logic calculation-related functions. For details about how to compose the expression of a [custom aggregation](https://www.jinfonet.com/kbase/designer15up2/userguide/HTML/bv/bv_elmnt.htm?rhhlterm=%22custom%20aggregation%22&rhsyns=%20#CA), refer to Custom aggregations in the JReport Designer User’s Guide.

When using a custom aggregation in a data component, you need to make sure that the group objects referenced in the custom aggregation are also added as group by fields, column/row headers or chart axis fields in the data component.
