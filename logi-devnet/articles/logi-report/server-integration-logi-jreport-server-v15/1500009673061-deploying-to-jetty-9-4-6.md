# Deploying to Jetty 9.4.6

This topic introduces how to deploy Logi JReport Server Guide v15 Server to Jetty 9.4.6. The example directory paths listed below are based on Unix. The instructions are applicable to both Unix and Windows installations; however, the format of the paths for Windows would use the Windows format, that is, `C:\Logi JReport\Server` instead of `/opt/Logi JReport/Server`.

It is assumed that the Logi JReport Server WAR file Logi JReport.war is located in the `/opt/Logi JReport/Server/bin/distribute` directory. To create the WAR file refer to the instructions in [Building a WAR/EAR File to Include a Self-contained Logi JReport Server](https://devnet.logianalytics.com/hc/en-us/articles/1500009648502-Building-a-WAR-EAR-file-to-Include-a-Self-contained-Logi-JReport-Server-Guide-v15-Server).

**To deploy Logi JReport Server to Jetty 9.4.6:**

1. Remove or comment out the following codes in the web.xml of Logi JReport.war if you are integrating Logi JReport Server with Jetty in a non-remote environment:

   `<login-config>  
   <auth-method>BASIC</auth-method>  
   </login-config>`
2. Add Logi JReport.war to the `/opt/Jetty9.4.6/demo_base/webapps` directory.
3. Start Jetty 9.4.6 using command java -jar /start.jar.
4. Access Logi JReport Server using the following URLs:

   `http://<hostname>:8080/Logi JReport/jrserver  
    http://<hostname>:8080/Logi JReport/admin/index.jsp  
    http://<hostname>:8080/Logi JReport/jinfonet/index.jsp`
