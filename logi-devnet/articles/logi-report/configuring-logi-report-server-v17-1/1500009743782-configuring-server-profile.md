# Configuring Server Profile

This topic describes the three tabs in the server profile: Change Password, Customize Server Preferences, and Customize Profile, the options, and how you can search for options conveniently.

Administrators can define initial settings of Logi Report Server and customize some features by configuring the server profile for all users in the Administration > Server Profile > Customize Server Preferences and Customize Profile pages of the Server Console. Any users can also configure their own profiles in the My Profile page of the Server Console.

There are three subjects in the server profile: [Change Password](#Password), [Customize Server Preferences](#Server), and [Customize Profile](#Profile). The Change Password subject is available in the My Profile page only. You can change the password for logging onto Logi Report Server. The Customize Server Preferences subject contains options for customizing the Logi Report Server web pages and some server features and has three sub categories: [General](#General), [Export Formats](#Export), and [Advanced](#Advanced). The Customize Profile subject is for customizing common features of Page Report Studio, Web Report Studio, and JDashboard and their individual features and preferences as well.

In the My Profile > Customize Server Preferences and Customize Profile pages (excluding the Features sub tab in the Page Report Studio/Web Report Studio/JDashboard tab), you can search for the options you need using the Search box. Type the text you want to search for and the options whose key values contain the matched text will be listed (The key values of the options are defined using the <Value></Value> tags in the file **keywords.xml** in the `<install_root>\resources\server\languages\en\properties` and `<install_root>\resources\server\languages\en-us\properties` directories. You can customize the key values to any text in any language). In the search result list, select an option and Server displays the page containing the option and highlights the matched text. When there are more than 10 results, you can select the **More** link at the end to show all the other results. After typing text in the Search box, you can also select ![Search Options](https://devnet.logianalytics.com/hc/article_attachments/4404885328919/btn_mvdown1.gif) that appears in the box to specify the following search options. To cancel the search operation, clear the text or select ![Delete button](https://devnet.logianalytics.com/hc/article_attachments/4404885308823/btn_dltobj.gif).

* **Highlight All**   
  Specifies whether to highlight all matched text.
* **Match Case**  
  Specifies whether to search for text that meets the case of the typed text.
* **Match Whole Word**   
  Specifies whether to search for text that matches the whole option name as defined in the <UIName></UIName> tags in keywords.xml. You can customize the <UIName></UIName> tags if you want. If selected, the key values of the options will be ignored.

You can also use [URL commands](https://devnet.logianalytics.com/hc/en-us/articles/1500009750382-Working-with-Server-via-URL#Preference) to change the user password or load the Server Preferences page directly.

| Enable Saving Parameter Values | Enables saving parameter values for reuse next time when specifying parameter values in order to run reports or dashboards. For a report/dashboard that has multiple parameters, the values of all the parameters specified at a time will be saved together as a group for the report/dashboard. The group is also called a list. Selecting a group as the value will apply the values in the group to all the corresponding parameters in the report/dashboard.  When the Yes check box is selected, you need to choose a way of saving the parameter values: Manually or Automatically. When switching from one to the other and OK is clicked, all saved parameter values for the current user will be cleared.   * **Manually**  If selected, it is up to users to manually save the parameter values they are going to reuse frequently and manage the saved values. Selecting a group from any of the parameters' value lists will apply the saved group of values to all the parameters. In addition, administrators can customize the allowed image types when report users insert images from the local file system into reports or dashboards in Page Report Studio, Web Report Studio, and JDashboard. To do this, in the Server Console, point to **Administration** on the system toolbar, and then select **Configuration** > **Upload** from the drop-down menu. Server displays the Upload page. Select the required image types and then select **Save**. The setting takes effect after Logi Report Server restarts.
