# Edit User Dialog

The Edit User dialog helps you to edit the user when editing a [security entry](https://devnet.logianalytics.com/hc/en-us/articles/1500010063241-Record-Level-and-Column-Level-Security#Data) or [business view security](https://devnet.logianalytics.com/hc/en-us/articles/1500010028582-Business-View-Security#Configure). It appears when you select a user and then select ![Edit button](https://devnet.logianalytics.com/hc/article_attachments/4404901116439/btn_edit.gif) in the [Security](https://devnet.logianalytics.com/hc/en-us/articles/1500010067681-Security-Dialog) dialog, [Edit Business View Security](https://devnet.logianalytics.com/hc/en-us/articles/1500010065501-Edit-Business-View-Security-Dialog) dialog or in the Security tab of the following dialogs: [Category Property](https://devnet.logianalytics.com/hc/en-us/articles/1500010065261-Category-Property-Dialog), [New View Element](https://devnet.logianalytics.com/hc/en-us/articles/1500010067281-New-View-Element-Dialog) or [Edit View Element](https://devnet.logianalytics.com/hc/en-us/articles/1500010065781-Edit-View-Element-Dialog).

The dialog contains two tabs: [General](#General) and [Relation](#Relation).

**OK**

Accepts all changes and closes the dialog.

**Cancel**

Does not retain any changes and closes the dialog.

**Help**

Displays the help document about this feature.

## General

Specifies the user information.

![Edit User dialog - General](https://devnet.logianalytics.com/hc/article_attachments/4404901265559/edtuser_gen.gif)

**User**

Specifies the name of the user.

**Source**

Displays the source of the user. It is read only.

**E-mail**

Specifies the e-mail address of the user.

**Description**

Specifies the description of the user.

## Relation

Specifies the relationship of the user and the existing roles and groups.

![Edit User dialog - Relation](https://devnet.logianalytics.com/hc/article_attachments/4404901265815/edtuser_rel.gif)

**Roles**

All available roles will be listed here. If a role is checked, the user will belong to the role.

**Groups**

All available groups will be listed here. If a group is checked, the user will belong to the group.
