# Report Object Properties

Designer displays all the objects a report contains in a tree structure in the [Report Inspector](https://devnet.logianalytics.com/hc/en-us/articles/5735570047895-Inspector-Panel). You can edit the object properties there to change the appearance of your report. This topic introduces the properties of the objects that a report may contain in alphabetical order, including their child objects if any.

![Note icon](https://devnet.logianalytics.com/hc/article_attachments/9898403122327/noteicon.jpg)Designer supports some objects in specific report type only, and for the same object, Designer provides some properties only when you use the object in certain report types: Page Report, Web Report, and Library Component. You can get details from the Available For column in the property table. In addition, page reports can use either query resources (Query Page Report) or business views (BV Page Report), and you can specify some properties for page reports using a specific data resource type only. When a property is available for page reports using both data resource types, you see Page Report in the Available For column; if it is available only for page reports using a specific data resource type, you see Query Page Report or BV Page Report in the Available For column.
