# Managing Logi Report Server

This topic discusses the management of resources, versions, tasks, server data, cached report data, in-memory cubes, and triggers on Logi Report Server. After going through this topic, you will be able to manage your Logi Report Server more effectively.

The following topics provide more information on management:

* [Managing Resources](https://devnet.logianalytics.com/hc/en-us/articles/1500009750041-Managing-Resources)
* [Managing Versions](https://devnet.logianalytics.com/hc/en-us/articles/1500009750181-Managing-Versions)
* [Managing Tasks](https://devnet.logianalytics.com/hc/en-us/articles/1500009750161-Managing-Tasks)
* [Managing Server Data](https://devnet.logianalytics.com/hc/en-us/articles/1500009723022-Managing-Server-Data)
* [Managing Cached Report Data](https://devnet.logianalytics.com/hc/en-us/articles/1500009750001-Managing-Cached-Report-Data)
* [Managing In-Memory Cubes](https://devnet.logianalytics.com/hc/en-us/articles/1500009750021-Managing-In-memory-Cubes)
* [Managing Triggers](https://devnet.logianalytics.com/hc/en-us/articles/1500009723162-Managing-Triggers)

**Note:** A [Logi Report Live license for Logi Report Server](https://devnet.logianalytics.com/hc/en-us/articles/1500009749561-Logi-Report-licenses#ServerLive) is required in order to use web report related features, and a [JDashboard license](https://devnet.logianalytics.com/hc/en-us/articles/1500009749561-Logi-Report-licenses#JDashboard) for dashboard and library component related features. If you do not have the licenses, contact your Logi Analytics account manager to obtain them.
