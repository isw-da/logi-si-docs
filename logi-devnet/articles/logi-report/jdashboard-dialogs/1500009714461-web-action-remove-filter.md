# Web Action - Remove Filter

The Web Action - Remove Filter dialog helps you to specify the filters you want to remove from a data component when it [receives a synchronization message](https://devnet.logianalytics.com/hc/en-us/articles/1500009712681-Synchronizing-the-Data-Components-in-a-Dashboard#Receive). This dialog appears when the web action \*Remove Filter is selected and you select ![Edit button](https://devnet.logianalytics.com/hc/article_attachments/4412139569559/btn_dshbrd_edt1.gif) in the Actions column of the [Receive Sync](https://devnet.logianalytics.com/hc/en-us/articles/1500009714301-Receive-Sync) dialog.

![Web Action - Remove Filter dialog](https://devnet.logianalytics.com/hc/article_attachments/4412131476247/wbactn_rmvfltr.gif)

**Remove Filters**

Removes the filter conditions from the data component, which are created via the Filter option on a field’s shortcut menu, the Filter web action and the Filter dialog.

**Remove \*On-screen Filters**

Removes the filter conditions from the data component, which are created by the On-screen Filter web action.

* **All Fields**  
   By default all filter conditions generated by the On-screen Filter web action will be removed. You can uncheck this option and then specify to remove filter conditions related to specific fields only.
* ![Add button](https://devnet.logianalytics.com/hc/article_attachments/4412112607767/btn_dshbrd_add1.gif)  
   Adds a condition line to remove the filter condition related to the specified field.
* ![Remove button](https://devnet.logianalytics.com/hc/article_attachments/4412139569815/btn_dshbrd_rmv1.gif)  
   Removes the selected condition line.

**OK**

Accepts the changes and closes this dialog.

**Cancel**

Does not retain any changes and closes the dialog.

![Help button](https://devnet.logianalytics.com/hc/article_attachments/4412139537431/btn_help.gif)

Displays the help document about this feature.

![Close button](https://devnet.logianalytics.com/hc/article_attachments/4412112572951/btn_close.gif)

Ignores the setting and closes this dialog.
