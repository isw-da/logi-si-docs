# Configuring Server Profile

On the Logi JReport Administration > Profile page, administrators can define some initial settings of Logi JReport Server and customize some features by configuring the server profile. Each end user can also configure his own profile on the Logi JReport Console > Profile page. The difference lies in that any settings specified by an end user can only be applied to this user, while the settings specified by administrators on the Logi JReport Administration > Profile page will be applied to all users.

There are three subjects on the Profile page: [Change Password](#Password), [Customize Server Preferences](#Server) and [Customize Profile](#Profile). The Change Password subject is available on the Logi JReport Console page only. Each end user can change the password he uses for logging onto Logi JReport Server. The Customize Server Preferences subject contains options for customizing the Logi JReport Server web pages and some server features and is divided into three sub categories: [General](#General), [Export Formats](#Export)  and [Advanced](#Advanced). The Customize Profile subject is for customizing common features of Page Report Studio, Web Report Studio, and JDashboard and their individual features and preferences as well.

You can also use [URL commands](https://devnet.logianalytics.com/hc/en-us/articles/1500009650542-Working-with-Logi-JReport-Server-Guide-v15-Server#Preference) to change the password of a user or load the Server Preferences page directly.

If Report Studio is selected, page reports will be opened in Page Report Studio, and web reports in Web Report Studio. If the property Default Format for Viewing Report of a page report tab or web report has been set to a specific format when the report is designed in Logi JReport Designer, the value in Designer has higher priority than the one here. |
| Default View for Page Report Studio | Specifies the default view of Page Report Studio when running page report in Page Report Studio by selecting report names or via URL:  * **Basic View**  If selected, Page Report Studio will run in Basic View by default. Users can switch to Interactive View if the link is available. * **Basic View Only**  If selected, Page Report Studio will run in Basic View only. Users are not able to switch the view. * **Interactive View**  If selected, Page Report Studio will run in Interactive View by default. Users can switch to Basic View if the link is available. |
| Default Mode for Web Report Studio | Specifies the default mode of Web Report Studio when running web reports in Web Report Studio by selecting report names or via URL:  * **View Mode**  If selected, Web Report Studio will run in View Mode by default. Users can switch to Edit Mode if the link is available. * **View Mode Only**  If selected, Web Report Studio will run in View Mode only. Users are not able to switch the mode. * **Edit Mode**  If selected, Web Report Studio will run in Edit Mode by default. Users can switch to View Mode if the link is available. |
| Default Mode for JDashboard | Specifies the default mode for viewing dashboards in JDashboard. You can choose between View Mode and Edit Mode. |
| Use Major Sort as Default | Specifies whether to use major sort as the default sort rule when sorting summary tables in Web Report Studio or JDashboard, which is to sort data by the last sort condition only. If unchecked, minor sort will be applied, which is to sort lower groups based on the sort result of all its higher level groups. |
| Open All Reports and Dashboards in New Window | Specifies whether to pop up Web Report Studio, Page Report Studio, or JDashboard in a new window. If unchecked, they will be embedded in the current window. |
| Use Wizard for Web Report Studio | Web Report Studio provides two ways for creating and working with web reports: a standard way using a wizard (when the Yes option is selected) and a quick start way to create a table report (when the Yes option is not selected). Select [here](https://devnet.logianalytics.com/hc/en-us/articles/1500009675101-Web-Report-Studio#Way) for more information. |
| Only Display CSS Styles in Style List | Specifies whether to display only the CSS styles in the style list. |
| Display the Last Login Time | Specifies whether to display the last login time of a user on the top banner of Logi JReport Server. If checked, the login time will be recorded in the login.properties file in `<install_root>\bin` after Logi JReport Server shuts down. |
| Display the Last Logout Time | Specifies whether to display the last logout time of a user on the top banner of Logi JReport Server. If checked, the logout time will be recorded in the logout.properties file in `<install_root>\bin` after Logi JReport Server shuts down. |
| Keep Completed Tasks for | Specifies for how many days Logi JReport Server will keep the completed tasks in the Completed list. If the value is set to 0, the server will keep the completed tasks until you delete them from the Completed page. |
| Parameter Display Size | Specifies the display length in characters of the parameters that are to be applied to both the completed and active schedules. |
| Folder Selector Type | Specifies the type of the tool used for selecting folder paths from the file system. This option is available on the Internet Explorer browser and applied to local published reports only. There are three types of folder selectors:   * **JavaScript format**  Folder selector is a dialog generated by JavaScript. * **VBScript format**  Folder selector is a dialog generated by VBScript. * **JSP format** Folder selector is an HTML page generated by JSP. This format is available to administrators only. |
| Skin Format | Specifies the skin format applied to Logi JReport Server and Page Report Studio UI. You can choose from Standard, Classical and Windows XP. |
| Columns Shown in \_ List | Specifies which columns will be displayed as default in the following list when you log onto Logi JReport Server: Resources, Scheduled, Running, Completed, Background Tasks, Report Result Versions, Report Versions, Catalog Versions, and Result Versions. |
| Show Loading Icon When Refreshing | Specifies whether to show the loading icon when refreshing reports or dashboards. |
| Enable IE Navigate2 | Specifies whether to load resources with IE Navigate2 method and its flags setting when viewing reports or loading URL links in Internet Explorer. |
| Export Formats | | | |
| Logi JReport Result | Specifies the default properties for exporting a report to a result file.  * **Zip Result**   Specifies whether to compress the result and make its size smaller. |
| Page Report Result | Specifies the default properties for exporting a report in the Page Report Result format.  * **Resolution**  Specifies the resolution of the result to zoom in/out, in DPI. The default value is obtained from the operation system, which is the resolution of your monitor, for example, 72 DPI on Unix or 96 on Windows. You can set higher/lower values to zoom in/out. |
| HTML | Specifies the default properties for exporting a report in the HTML format. For details about properties of this format, select [here](https://devnet.logianalytics.com/hc/en-us/articles/1500009670701-Advanced-Run#HTML). |

For details about other properties of this format, select [here](https://devnet.logianalytics.com/hc/en-us/articles/1500009670701-Advanced-Run#PDF). |

| Enable Waiting Page | Specifies whether to show the waiting pages. You can select the Customize button to customize waiting pages in the [Customize Waiting Page](https://devnet.logianalytics.com/hc/en-us/articles/1500009670761-Customize-Waiting-Page) dialog. Available to administrators on the Logi JReport Administration > Profile page only. |
| Specify Default Language | Specifies an environment language for Logi JReport Server. All UI text and messages will be displayed in the specified language. The available languages come from the qualified language packages defined in `<server_install_root>\resources\server\languages`. By default there are two built-in English language packages: en and en-us. You can add more language packages following the section [NLS at Application Level](https://devnet.logianalytics.com/hc/en-us/articles/1500009673861-NLS-at-Application-Level).  * **Language**  Specifies the environment language of Logi JReport Server. |

| Specify Date Time Format | Specifies the default date and time formats in which such values will be displayed on the Logi JReport Server web pages, for example, the last modified time of a report, the time when a schedule take is finished. |

| Identify Server Preference | Specifies whether a task can be performed on a manually specified clustered server when scheduling a report. Available only in cluster environment. |
| Use Accessible Version | Sets the accessible version of Logi JReport Server as the default portal instead of Logi JReport Console page. See [Accessibility](https://devnet.logianalytics.com/hc/en-us/articles/1500009644202-Accessibility) for more information. |
| Enable Saving Parameter Values | Enables saving parameter values for reuse next time when specifying parameter values in order to run reports. For a report that has multiple parameters, the values of all the parameters specified at a time will be saved together as a group for the report. The group is also called a list. Selecting a group as the value will apply the values in the group to all the corresponding parameters in the report.  When the Yes checkbox is selected, you need to choose a way of saving the parameter values: Manually or Automatically. When switching from one to the other and OK is selected, all saved parameter values for the current user will be cleared.   * **Manually**  If checked, it is up to users to manually save the parameter values they are going to reuse frequently and manage the saved values. Then in some places where users are asked to specify parameter values, such as advanced running and scheduled running, the options for using saved parameter values, saving the specified values, and deleting the saved values will be provided. * **Automatically**  If checked, Logi JReport saves user submitted parameter values on a user-report or user-dashboard basis. The automatically saved parameter value groups will be available on the value drop-down list of each of those parameters next time when the same user is asked to specify parameter values for the same report. Selecting a group from any of the parameters' value lists will apply the saved group of values to all the parameters. A group on the value lists is a one-row values' set: it lists the values of the parameters in one row with blank to separate between parameters and the values are in an order same as the display order of the parameters. * **Maximum Number of Auto Complete Parameters List**  For both manually and automatically saved parameter values groups or lists for a report, there is a number restriction. By default it is 3, which means the manually saved lists for a report or the auto saved lists for a report can be no more than three. When the saved lists exceed the maximum number or if you change the current number to a smaller one, the oldest lists will be removed. |

| Save Sort Criteria | Specifies whether to check the Save Sort Criteria option by default in the save dialog when saving a report/dashboard to a report/dashboard version or as a new report/dashboard, or whether to save the sort defined in a report/dashboard by default when saving an existing report/dashboard. |
| Enable Partial Report Result | Specifies whether to retrieve partial data only when running reports in Page Report Studio and Web Report Studio. When a report contains a large amount of data, it will be helpful to retrieve only a specific number of records for performance concern. When this option is checked, a report data mode drop-down list is available on the toolbar of Page Report Studio and Web Report Studio, with which end users can switch to show full data of a report or only get partial report results.  * **Default Setting**  Specifies the default report data mode to retrieve data: Partial Data or Full Data. In addition, administrators can customize the allowed image types when end users insert images from the local file system into reports or dashboards in Page Report Studio, Web Report Studio and JDashboard. To do this, on the Logi JReport Administration page, select **Configuration** on the system toolbar and then select **Upload** from the drop-down menu. On the Configuration - Upload page, check the required image types and select **Save**. The setting takes effect after Logi JReport Server is restarted.
