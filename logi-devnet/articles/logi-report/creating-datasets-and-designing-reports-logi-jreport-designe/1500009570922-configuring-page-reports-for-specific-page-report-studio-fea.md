# Configuring Page Reports for Specific Page Report Studio Features

Page reports developed by Logi JReport Designer can be published to a standalone Logi JReport Server or to web applications integrated with Logi JReport Server. After that, they can be viewed and analyzed in Page Report Studio. Certain Page Report Studio features require a page report to be pre-configured in Logi JReport Designer before being published to Logi JReport Server.

The following topics provide information about tasks associated with page report studio features:

> * [Obtaining Detailed Information from a Banded Object](https://devnet.logianalytics.com/hc/en-us/articles/1500009592841-Obtaining-Detailed-Information-from-a-Banded-Object)
> * [Customizing the Field Display Names](https://devnet.logianalytics.com/hc/en-us/articles/1500009592861-Customizing-the-Field-Display-Names)
> * [Predefining Sort and Filter Conditions](https://devnet.logianalytics.com/hc/en-us/articles/1500009570962-Predefining-Sort-and-Filter-Conditions)
> * [Enabling Sort and Filter on Labels](https://devnet.logianalytics.com/hc/en-us/articles/1500009592881-Enabling-Sort-and-Filter-on-Labels)
> * [Setting Filter Options for an Object](https://devnet.logianalytics.com/hc/en-us/articles/1500009570942-Setting-Filter-Options-for-an-Object)

**Tip:** In addition to the above features, Logi JReport allows you to create a table of contents for Page Report Studio use by setting the TOC Anchor and Anchor Display Value properties for each component in a report, then in Page Report Studio, you can use the TOC Browser to locate the component. For details about how to show components in a TOC tree, see [Showing components in a TOC tree](https://devnet.logianalytics.com/hc/en-us/articles/1500009570982-Editing-a-Page-Report#TOC).
