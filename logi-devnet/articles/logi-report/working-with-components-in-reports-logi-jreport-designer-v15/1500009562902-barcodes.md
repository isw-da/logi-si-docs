# Barcodes

A barcode can be used as a component and directly inserted into a page report that is created using a query resource.

1. Position the mouse pointer at the destination where you want to insert the barcode. A barcode can be inserted in the report areas listed in [Component placement](https://devnet.logianalytics.com/hc/en-us/articles/1500009583241-Working-with-Components-in-Reports#Relationship).
2. Select **Insert** > **Barcode**, or **Home** > **Insert** > **Barcode**. The [Barcode Expert](https://devnet.logianalytics.com/hc/en-us/articles/1500009585741-Barcode-Expert-Dialog) dialog appears.

   ![Barcode Expert dialog](https://devnet.logianalytics.com/hc/article_attachments/4404889390487/brcd.gif)
3. Select a dataset from the Barcode Resources drop-down list, and then specify the source of the barcode: a DBField, formula, summary, or parameter.
4. Select the required barcode type from the Symbology drop-down list. You can preview the format of the selected symbology in the Sample box (for details about barcode symbologies, see [Appendix 1: Barcode Symbologies](https://devnet.logianalytics.com/hc/en-us/articles/1500009562562-Appendix-1-Barcode-Symbologies)).
5. Select the unit for the values of the quiet zone, narrow width, supplement, height and ratio from the Scale Mode drop-down list.
6. Specify the space around the barcode in the Quite Zone field (most symbologies require quiet zones that precede and follow the barcode).
7. Set the barcode bar width in the Narrow Width field.
8. Specify the supplement of the barcode if necessary. If you select the barcode type as Code 39, Code 128/128A/128B/128C or Codabar, you will not need to set the supplement.
9. Set the height of the barcode bars in the Height field.
10. Specify the width ratio of the thick bar to the thin bar in the Ratio field if you selected the barcode type as Code 39 or Codabar. The ratio can only be 2.0 or 3.0. Any ratio value not equal to 2.0 or 3.0 will be regarded as 2.0.
11. Change the default value of the barcode in the Default Message field if necessary.
12. Select the **Enable Checking Digits** checkbox if you want to include check digits in the barcode.
13. If you have selected the barcode type as Code 39, Code 128/128A/128B/128C, or Codabar, then you can check **Display HR** to display the barcode numbers together with the barcode.
14. Upon finishing, select **Insert** to close the dialog, then select the mouse button in the location where you want the barcode to be displayed.

**Notes:**

* Where a barcode can be inserted in a report depends on what you have selected as its source.
* You can also create a barcode component by first inserting a DBField, formula, summary or parameter and then [changing its display type to barcode](https://devnet.logianalytics.com/hc/en-us/articles/1500009563422-Changing-the-Display-Type-of-a-Label#Barcode).

**See an example:** The SampleComponents catalog, included with Logi JReport Designer, contains reports that have examples of how each component type could be used in a report. For the barcode component example, open `<install_root>\Demo\Reports\SampleComponents\Barcode.cls`.
