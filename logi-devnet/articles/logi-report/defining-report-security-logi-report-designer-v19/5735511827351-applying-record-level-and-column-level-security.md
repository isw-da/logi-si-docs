# Applying Record Level and Column Level Security

You can use Record Level Security (RLS) and Column Level Security (CLS) to control user access to different subsets of data and ensure that people only see what they are supposed to see: RLS enables you to define which records you want to reveal to any given user; CLS enables you to define which report column you want to reveal to any given user. No matter to whom you need to provide information, a plant manager or thousands of customers, RLS and CLS enable you to provide different users with accordingly different, but appropriate content.

In Designer, you can define RLS either based on a catalog data source, or on a single report, or simultaneously on both. You can specify CLS on data source scope only. This topic introduces how you can set RLS and CLS at different scopes and apply them to secure your reports.
