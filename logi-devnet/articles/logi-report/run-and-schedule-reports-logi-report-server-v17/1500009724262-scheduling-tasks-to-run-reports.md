# Scheduling Tasks to Run Reports

By setting up schedule tasks on reports, you can make reports run automatically at one or more designated dates and times, and then publish the report results to different destinations in various formats such as PDF, HTML, Excel, and so on. Scheduled reports can greatly improve the performance of the reporting service in an organization, for example, reports can be run during low network traffic times. Scheduled reports can also capture a time period exactly, for example the last day of the quarter.

Logi Report Server supports delivering report results to the following destinations: To Version, To Disk, To E-mail, To Printer, To Fax, and To FTP. You can create a schedule task on one report to publish its result to all of the destinations at a time, however when a schedule task is based on multiple reports, only To Version and To E-mail are supported.
