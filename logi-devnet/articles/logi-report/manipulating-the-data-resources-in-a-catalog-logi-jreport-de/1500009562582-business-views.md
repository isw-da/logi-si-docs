# Business Views

Business views provide report developers and end users with a business-oriented view of their data. They internally contain database connections and relationships between view elements that are required for creating reports yet display the available data in an easy to use flat structure for creating reports. All reports created from business views are automatically interactive with the end user able to dynamically modify the report such as drill down, drill up and even add and remove fields from the report.

Business views shield report end users from having to understand physical structure, and enable the easy building of complex reports containing multiple components. Business views enable report developers and end users to build reports and analyze data based on a set of view elements that they can easily understand. For example, they do not need to worry about where the data source is from (SQL, stored procedure, view, etc.), or what the data source is (Oracle, Sybase, XML, etc.). Business views also enable IT professionals to maintain control of business data and to ensure its integrity, while presenting end users with an intuitive view of the underlying data.

A business view can mash up multiple data resources from different data source connections and distributed joins need to be set up among the data resources. Data mashup makes it possible that multiple separated application systems in your enterprise can work together to provide data to reports. The performance depends on that of the computer hardware running Logi JReport. Generally speaking, mixed data sources cannot get the performance optimization as single data sources do from [Push Down Group Query](https://devnet.logianalytics.com/hc/en-us/articles/1500009590441-JDBC-HIVE-Connection-Properties#PushDown). However by working with Logi JReport Server's In-memory Cube, better performance is guaranteed even with big data retrieve.
