# Creating and Managing Catalogs

A catalog in Logi Report is a report and metadata repository. This topic introduces the basics and advantages of the Logi Report catalogs and how you can create and manage catalogs in Designer.

A catalog is represented at the operating system level as a folder that contains the corresponding catalog file and the other report object files. Every report must exist within the context of a catalog folder and must be tied to a catalog file which is a physical file that stores data resources and object definitions. You can apply a catalog file for one report or multiple reports in the same catalog folder. Designer stores any other resources such as images used by the reports in the same catalog folder.

Designer comes with four catalog folders: JinfonetGourmetJava, SampleComponents, SampleReports, and TutorialReports. The catalog files in the JinfonetGourmetJava and TutorialReports folders are used for the Logi Report Tutorial manual. You can get samples from any of them to learn Designer.
