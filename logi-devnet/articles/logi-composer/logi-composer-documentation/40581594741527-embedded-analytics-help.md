# Embedded Analytics Help

Looking for help with your Embedded Analytics Product?

| Area of Assistance | Help Topic |
| --- | --- |
| **Introduction**  Covers our four main value pillars — effortless authoring, embedded self-service, the z-Engine query engine with smart data connectors, and microservices architecture — along with links to get started, release notes, and system requirements.  Start here if you're new to our offering or need a quick orientation before diving into specific topics. | Composer v26 [link]  Embedded Analytics v26 [link] |
| **Manage Custom Charts**  Covers how to create, import, and manage custom chart types, with links to the CLI workflow, UI management, configuration properties, the API reference, and a step-by-step tutorial.  Start here if you're an admin looking to build or manage custom visuals. | Composer v26 [link]  Embedded Analytics v26 [link] |
| **How Dashboard Permissions are Determined**  Covers how dashboard permissions are resolved when tenant, group, and user levels conflict, using a "most permissive" model where users get the highest access granted at any level. Also explains how source-level permissions affect what data users can see on a dashboard, and how permissions work for imported objects.  Start here if you need to understand how Composer resolves conflicting access levels across tenants, groups, and users for a given dashboard. | Composer v26 [link]  Embedded Analytics v26 [link] |
| **Embed Components Into Your Application**  Covers how to embed dashboards, visual authoring tools, and the source editor into your application, including what controls end-user interactivity — embed mode, visual/dashboard interactivity settings, and editor placement. Links to the JavaScript API reference, prerequisites, HTML snippet generation, and Trusted Access setup.  Start here if you're a developer ready to integrate embedded components into your app. | Composer v26 [link]  Embedded Analytics v26 [link] |
| **About Visual Permissions**  Covers how to grant read, write, or delete access to visuals at the tenant, group, or user level using a most-permissive model, including who's allowed to manage those permissions and the constraint that users can only assign permissions equivalent to their own.  Start here if you need to configure who can see, edit, or delete specific visuals in your environment. | Composer v26 [link]  Embedded Analytics v26 [link] |
| **About Source Permissions**  Covers Data Access, Read, Write, and Delete permissions for data sources at the user, group, and tenant level, including the distinction between Data Access and Read permission, privilege requirements for managing permissions, and how row/column security filters interact with source admin roles.  Start here if you're responsible for configuring who can access or modify specific data sources in your environment. | Composer v26 [link]  Embedded Analytics v26 [link] |
