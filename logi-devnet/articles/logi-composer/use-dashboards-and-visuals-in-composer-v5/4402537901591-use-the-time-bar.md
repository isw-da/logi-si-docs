# Use the Time Bar

# Use the Time Bar

Composer supports time-based attributes for both live streaming sources (like sources created using upload API) and historical data sets (like SQL-based tables). When time-based attributes are available for a data source, the time bar can be used on a dashboard.

Use the time bar to filter your visuals using a time attribute. It appears at the bottom of your dashboard. The time bar that is shown depends on the visual on the dashboard that you have selected. The following diagram depicts a sample time bar, with its parts identified:

![](https://devnet.logianalytics.com/hc/article_attachments/4406763928343/time-bar-layout_672x99.png)

Each section is described in the following table:

| Number | Picture | Description |
| --- | --- | --- |
| 1 |  | Shows the time field selected for the time bar. Use this to change the time field. See [*Changing the Time Bar Field*](https://devnet.logianalytics.com/hc/en-us/articles/4402537899671-Changing-the-Time-Bar-Field). |
| 2 |  | Allows you to select a static or dynamic time range for the time bar. See [*Adjusting the Time Bar Range Using the Time Range Dialog*](https://devnet.logianalytics.com/hc/en-us/articles/4402537900311-Adjusting-the-Time-Bar-Range-Using-the-Time-Range-Dialog). |
| 3 |  | The From slider. Slide this to the right to change the minimum static time range. See [*Adjusting the Time Bar Range Using Time Bar Sliders*](https://devnet.logianalytics.com/hc/en-us/articles/4402552906903-Adjusting-the-Time-Bar-Range-Using-Time-Bar-Sliders). |
| 4 |  | The To slider. Slide this to the left to change the maximum static time range. See [*Adjusting the Time Bar Range Using Time Bar Sliders*](https://devnet.logianalytics.com/hc/en-us/articles/4402552906903-Adjusting-the-Time-Bar-Range-Using-Time-Bar-Sliders). |
| 5 |  | Indicates whether or not live data is [enabled](https://devnet.logianalytics.com/hc/en-us/articles/4402537788055-Configuring-Time-Bar-Defaults) for the visual's data source. If **Live** appears here, it is live data is enabled. If **Max** appears here, live data is not enabled. |
| 6 |  | Allows you to control data playback, if it is [enabled](https://devnet.logianalytics.com/hc/en-us/articles/4402537788055-Configuring-Time-Bar-Defaults) for the visual's data source. |
| 7 |  | When you select this, the time bar zooms to view only the specified range. All other times are not visible. See [*Zooming Into a Time Bar Range*](https://devnet.logianalytics.com/hc/en-us/articles/4402552907671-Zooming-Into-a-Time-Bar-Range). |
| 8 |  | When you select this, the time bar expands to show the available range of times in the data. See [*Zooming Into a Time Bar Range*](https://devnet.logianalytics.com/hc/en-us/articles/4402552907671-Zooming-Into-a-Time-Bar-Range). |
| 9 |  | When you select this, the maximum range of data is selected. The data shown expands to include all dates and times in the data. See [*Selecting the Maximum Time Range*](https://devnet.logianalytics.com/hc/en-us/articles/4402537900951-Selecting-the-Maximum-Time-Range). |

You can set time bar settings in data source configurations that will be used, by default, for each new visual created using the data source. See [*Configuring Time Bar Defaults*](https://devnet.logianalytics.com/hc/en-us/articles/4402537788055-Configuring-Time-Bar-Defaults).
