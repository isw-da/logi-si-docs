# Sources Page

# Sources Page

The **Sources** page allows you to review and maintain your Composer data source configuration definitions.

![](https://devnet.logianalytics.com/hc/article_attachments/4406747382295/noteicon.jpg) All users can view the **Sources** page. However, if you are logged in as a user who has not been granted [permissions](https://devnet.logianalytics.com/hc/en-us/articles/4405859328535-About-Data-Source-Permissions) for any data sources, the **Sources** page will display "No sources are available." If you are a user who has only **read**[permissions](https://devnet.logianalytics.com/hc/en-us/articles/4405859328535-About-Data-Source-Permissions) for a data source, the data source is shown on this page, but the Connection column will be blank.However, if you are a user who belongs to a group that is granted the **Can Administer Sources**[privilege](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference), you can add and maintain data source configurations from this page. In addition,

![](https://devnet.logianalytics.com/hc/article_attachments/4406743806999/sources-page-permissions_757x296.png)

The Sources page includes the following features:

1. The ![](https://devnet.logianalytics.com/hc/article_attachments/4406743807255/import-source-btn.png) button allows you to import a data source configuration from another Composer instance. See [*Import a Data Source Configuration*](https://devnet.logianalytics.com/hc/en-us/articles/4405859319191-Import-a-Data-Source-Configuration).
2. The ![](https://devnet.logianalytics.com/hc/article_attachments/4406743807639/add-source-btn.png) button allows you to add a new data source configuration. See [*Define a Data Source Configuration*](https://devnet.logianalytics.com/hc/en-us/articles/4405859320727-Define-a-Data-Source-Configuration).
3. The table lists and allows you to maintain your previously defined data source configurations. You can add edit, enable, disable, download, and delete data source configurations in this table.
4. A search bar at the top of the page that you can use to search for a data source configuration in the table.

To access the Sources page:

1. Log into Composer as an administrator or as a user with the **Can Create new Data Sources**[privilege](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference).
2. Select **Sources** on the [UI menu](https://devnet.logianalytics.com/hc/en-us/articles/4405859444119-The-Composer-UI-Menu) (![](https://devnet.logianalytics.com/hc/article_attachments/4406743720343/hamburger.png)) or the [top-level navigation menu](https://devnet.logianalytics.com/hc/en-us/articles/4405851152407-The-Top-Level-Navigation-Banner), or select the **Sources** box on the [Home page](https://devnet.logianalytics.com/hc/en-us/articles/4405851121559-Home-Page). The [Sources](#) page appears.

You can edit, enable, disable, download, and delete data source configurations listed in the table on the Sources page. You can also clear the cache for a data source configuration. If there are many sources listed, you may need to search for the data source configuration you need. Use the search bar to search for a source definition in the list. See [Search for a Definition in a List](https://devnet.logianalytics.com/hc/en-us/articles/4405850911127-Search-for-a-Definition-in-a-List).

Each column in the table is described below. The Enabled, Type, Name, Connection, Author, and Modified Date columns are sortable.

| Column | Description |
| --- | --- |
| Enabled | A switch you can use to enable a data source configuration. After its initial definition, a data source configuration is enabled by default. Data source configuration must be enabled before visuals can be created using them. See [*Enable a Data Source Configuration*](https://devnet.logianalytics.com/hc/en-us/articles/4405859316119-Enable-a-Data-Source-Configuration).  Data source configurations can no longer be disabled. Once they are enabled, they cannot be disabled by anyone. You can control access to a data source configuration using [row](https://devnet.logianalytics.com/hc/en-us/articles/4405859333655-Restrict-Access-to-Data-Using-Row-Security) and [column](https://devnet.logianalytics.com/hc/en-us/articles/4405851034007-Restrict-Access-to-Fields-Using-Column-Security) data source restrictions, or by [deleting](https://devnet.logianalytics.com/hc/en-us/articles/4405859314967-Delete-a-Data-Source-Configuration) the data source from the system. |
| Type | An icon identifying the data store type for the data source configuration. |
| Name | The name assigned the data source configuration. For flat file data sources, you can upload additional files or clear the data. |
| Connection | The [data store connection definition](https://devnet.logianalytics.com/hc/en-us/articles/4405850909079-Manage-Data-Store-Connections) used to connect to the external data store. |
| Author | The name of the user who defined the data source configuration definition. |
| Modified Date | The time stamp when the data source configuration definition was last modified. |
| Permissions | The  button in this column allows you to assign and manage permissions for a data source configuration. You can only define permissions for a data source if you are logged in as an administrator, a user with the **Can Administer Sources**[privilege](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference), a user with the **Can Manage Source Permissions** privilege, or a user with **write**[permission](https://devnet.logianalytics.com/hc/en-us/articles/4405859328535-About-Data-Source-Permissions) for the data source. |
| Row | The  button in this column allows you to define row security for the data source for different [authorization groups](https://devnet.logianalytics.com/hc/en-us/articles/4405850856599-Manage-Group-Definitions). You can only define row security for a data source if you are logged in as an administrator, a user with the **Can Administer Sources**[privilege](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference), a user with the **Can Manage Source Permissions** privilege, or a user with **write**[permission](https://devnet.logianalytics.com/hc/en-us/articles/4405859328535-About-Data-Source-Permissions) for the data source. |
| Column | The  button in this column allows you to define column security for the data source for different [authorization groups](https://devnet.logianalytics.com/hc/en-us/articles/4405850856599-Manage-Group-Definitions). You can only define column security for a data source if you are logged in as an administrator, a user with the **Can Administer Sources**[privilege](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference), a user with the **Can Manage Source Permissions** privilege, or a user with **write**[permission](https://devnet.logianalytics.com/hc/en-us/articles/4405859328535-About-Data-Source-Permissions) for the data source. |
| Actions | Shows icons you can select to perform actions against the data source.   * Select the  icon to download (export) the data source configuration as a JSON file. See [*Export a Data Source Configuration*](https://devnet.logianalytics.com/hc/en-us/articles/4405851025559-Export-a-Data-Source-Configuration). This option is available for all data source types except flat files. You can only export a data source if you are logged in as an administrator, a user with the **Can Administer Sources**[privilege](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference), or a user with **read**[permission](https://devnet.logianalytics.com/hc/en-us/articles/4405859328535-About-Data-Source-Permissions) for the data source. * Select the  icon to delete a data source configuration. Before you delete a data source configuration, you must delete all the dashboards that use it. See [*Delete a Data Source Configuration*](https://devnet.logianalytics.com/hc/en-us/articles/4405859314967-Delete-a-Data-Source-Configuration). You can only delete a data source if you are logged in as an administrator, a user with the **Can Administer Sources**[privilege](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference), or a user with **delete**[permission](https://devnet.logianalytics.com/hc/en-us/articles/4405859328535-About-Data-Source-Permissions) for the data source. * Select the  icon to clear the data cache for this data source configuration. This clears both the visual (visual data) cache as well as the metadata (field statistics) cache. See [*How Composer Caches Data*](https://devnet.logianalytics.com/hc/en-us/articles/4405859286935-How-Composer-Caches-Data) and [*Clear the Cache for a Data Source Configuration*](https://devnet.logianalytics.com/hc/en-us/articles/4405859313815-Clear-the-Cache-for-a-Data-Source-Configuration). You can only clear the cache for a data source if you are logged in as an administrator, a user with the **Can Administer Sources**[privilege](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference), or a user with **read**[permission](https://devnet.logianalytics.com/hc/en-us/articles/4405859328535-About-Data-Source-Permissions) for the data source. * Select the  icon to view the More menu for a data source configuration.
