# Refresh Tab

# Refresh Tab

Composer maintains data source metadata and, optionally, a cached result set of the data from the data store for each data source configuration you define. Using the Refresh tab, Composer administrators can define refresh jobs for this cached data in the data source configuration. By default no refresh job is defined (**No Schedule** is selected).

![](https://devnet.logianalytics.com/hc/article_attachments/4406743835671/refresh-tab_672x274.png)

You can refresh the entire data source, all the fields in a data source, or select fields in a data source. For more information, see the following sections:

* [*Trigger Refresh Jobs*](https://devnet.logianalytics.com/hc/en-us/articles/4405851159063-Trigger-Refresh-Jobs)
* [*Set Up a Data Source Refresh Job*](https://devnet.logianalytics.com/hc/en-us/articles/4405851158167-Set-Up-a-Data-Source-Refresh-Job)
* [*Review Refresh Jobs*](https://devnet.logianalytics.com/hc/en-us/articles/4405851155095-Review-Refresh-Jobs)

When you are finished, select ![](https://devnet.logianalytics.com/hc/article_attachments/4406743836439/save-next-btn_78x24.png) to proceed to the [Visuals Tab](https://devnet.logianalytics.com/hc/en-us/articles/4405859325591-Visuals-Tab) in the data source configuration wizard. Select ![](https://devnet.logianalytics.com/hc/article_attachments/4406743834135/back-btn.png) to return to the [*Fields Tab*](https://devnet.logianalytics.com/hc/en-us/articles/4405851026455-Fields-Tab).

![](https://devnet.logianalytics.com/hc/article_attachments/4406747382295/noteicon.jpg) You must be logged in as an administrator or as a user with the [group privilege](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference)**Can Create new Data Sources** to see the Refresh tab of the data source configuration wizard.
