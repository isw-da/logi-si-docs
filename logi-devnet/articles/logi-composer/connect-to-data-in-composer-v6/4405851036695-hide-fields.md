# Hide Fields

# Hide Fields

By default, all fields in a data source are visible and available for use in [custom metrics](https://devnet.logianalytics.com/hc/en-us/articles/4405859285399-Maintain-Custom-Metrics), [derived fields](https://devnet.logianalytics.com/hc/en-us/articles/4405859299479-Maintain-Derived-Fields), and in your dashboard visuals. However, there may be source fields in the data that you prefer to hide. You can also hide [derived fields](https://devnet.logianalytics.com/hc/en-us/articles/4405859299479-Maintain-Derived-Fields).

Hidden source and derived fields are not available for use in [custom metrics](https://devnet.logianalytics.com/hc/en-us/articles/4405859285399-Maintain-Custom-Metrics) or dashboard visuals, but they can be used in [derived fields](https://devnet.logianalytics.com/hc/en-us/articles/4405859299479-Maintain-Derived-Fields).

* [*When to Hide Fields*](#When)
* [*How to Hide a Field*](#How)

## When to Hide Fields

Source and derived fields cannot be hidden if they are already used in any of the following:

* a custom metric
* a dashboard visual
* the default visual settings for a data source configuration.

Consequently, the best time to hide a field is when you set up a data source configuration or define a derived field for the first time, and before they can be used.

![](https://devnet.logianalytics.com/hc/article_attachments/4406747382295/noteicon.jpg) After you save a data source configuration, its default visual settings for each visual selected on the [Visuals tab](https://devnet.logianalytics.com/hc/en-us/articles/4405859325591-Visuals-Tab) are stored. All available (non-hidden) fields are automatically included in these default visual settings.

If you try to hide a field that is already used in a custom metric, visual, or default visual settings, error messages will display. You will have to remove the field first from all the places in which it is used before you can hide it.

## How to Hide a Field

Source fields and derived fields are hidden on the [Fields tab](https://devnet.logianalytics.com/hc/en-us/articles/4405851026455-Fields-Tab) of a data source configuration.

To hide a field:

1. Make sure you are logged in as an administrator or a user with the correct group privileges to manage the data source configuration.
2. Select **Sources** on the [UI menu](https://devnet.logianalytics.com/hc/en-us/articles/4405859444119-The-Composer-UI-Menu) (![](https://devnet.logianalytics.com/hc/article_attachments/4406743720343/hamburger.png)) or the [top-level navigation menu](https://devnet.logianalytics.com/hc/en-us/articles/4405851152407-The-Top-Level-Navigation-Banner), or select the **Sources** box on the [Home page](https://devnet.logianalytics.com/hc/en-us/articles/4405851121559-Home-Page). The [Sources](https://devnet.logianalytics.com/hc/en-us/articles/4405851037719-Sources-Page) page appears.
3. Select the **Fields** tab in the data source configuration wizard.
4. To hide a source field, clear the corresponding **Visible** checkbox in the Field table at the top of the [Fields tab](https://devnet.logianalytics.com/hc/en-us/articles/4405851026455-Fields-Tab). If the **Visible** checkbox is checked, a field is visible to users. If it is cleared (not checked), it is hidden.
5. To hide a derived field, clear the corresponding **Visible** checkbox in the Derived Field table in the middle of the [Fields tab](https://devnet.logianalytics.com/hc/en-us/articles/4405851026455-Fields-Tab). If the **Visible** checkbox is checked, a field is visible to users. If it is cleared (not checked), it is hidden.
6. Save the data source configuration.

   If a field cannot be hidden because it is used in a custom metric, dashboard visual, or in the data source configuration's default visual settings, error messages will display.
