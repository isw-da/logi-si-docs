# Manage Group Definitions

Group definitions assign [privileges](https://devnet.logianalytics.com/hc/en-us/articles/4405859114647-Group-Privilege-Reference) to groups of users. Groups are most useful when a number of Composer users require the same access restrictions. Users can be assigned to multiple groups.

![](https://devnet.logianalytics.com/hc/article_attachments/4406747382295/noteicon.jpg) SAML and LDAP groups that are automatically created in Composer must be manually assigned privileges.

Composer account administrators and users assigned to groups with the **Can Administer Groups** privilege can maintain group definitions. They can:

* Add, edit, or remove group definitions
* Assign and remove users in a group definition
* Authorize users in the group to perform specific Composer functions

If your user ID is not assigned the **Can Administer Groups** privilege or is not an administrator, you cannot assign groups to a user. In addition, only administrators can assign users to the **Administrators** group.

For more info about managing users, see [*Manage User Definitions*](https://devnet.logianalytics.com/hc/en-us/articles/4405859120023-Manage-User-Definitions).
