# Manage User Definitions

Composer provides the account management controls necessary to create user definitions and manage user access of Composer. Authorization for users to use product features and functions is controlled by the [groups](https://devnet.logianalytics.com/hc/en-us/articles/4402962859799-Manage-Group-Definitions) to which the users are assigned.

User definitions are managed by Composer administrators, users with the **Can Administer Users** group [privilege](https://devnet.logianalytics.com/hc/en-us/articles/4402955416087-Group-Privilege-Reference), and supervisors. All such users can add and remove user definitions, but only supervisors can assign accounts to users and only administrators (or users with the **Can Administer Groups** group [privilege](https://devnet.logianalytics.com/hc/en-us/articles/4402955416087-Group-Privilege-Reference)) can assign users to groups. There are some other differences. For example, only administrators and users with the **Can Administer Users** group [privilege](https://devnet.logianalytics.com/hc/en-us/articles/4402955416087-Group-Privilege-Reference) can reset a user's password.

User definitions can be manually created or [imported](https://devnet.logianalytics.com/hc/en-us/articles/4402962863383-Import-User-Definitions) via the LDAP security protocol. In addition, if using SAML single sign-on protocol, users and groups may be automatically provisioned in Composer (account level synchronization).
For more information, see [*Supported Authentication Tools*](https://devnet.logianalytics.com/hc/en-us/articles/4402955662103-Supported-Authentication-Tools).
