# Role Setup

# Role Setup

The **Role Setup** page allows user to manage roles.

## List and View Roles

1. In browser, log in to Izenda as a user with Role Setup permissions.
2. Click Settings, then Role Setup in the left menu.
3. Select the Setting Level: either System or a specific tenant.
4. List of roles and statuses will be displayed in the Middle Panel.
5. Click any role in the list.
6. The role’s information will be displayed in the 4 tabs Data Model Access, Permissions, Users, and Permission Summary.

[![../_images/Role_Setup_List_and_Details.png](https://devnet.logianalytics.com/hc/article_attachments/12528204670487/role_setup_list_and_details_787x393.png)](https://devnet.logianalytics.com/hc/article_attachments/12528220520983/role_setup_list_and_details.png)

## Search for Roles

The Search box at the top allows searching for specific roles.

1. Type a partial name and click the search icon (🔍).

   ![../_images/Role_Setup_Search.png](https://devnet.logianalytics.com/hc/article_attachments/12528220530455/role_setup_search_185x145.png)
2. Only matching roles will be displayed in the list.
3. Clear the search box to see the full list.

## Add Role

1. Click the + Add Role button at the top.
2. Enter a unique name for the role.
3. In Data Model Access tab, tick items in Data Sources to select data sources that the role can access.

   > It is possible to select only some fields of a table. In this case, users of the role will only see selected fields in the table.
   >
   > Tick **Show selected items only** checkbox to manage the list easily. This option is available from version 2.9.0.
   >
   > ![../_images/Role_Data_Model_Access.png](https://devnet.logianalytics.com/hc/article_attachments/12528204678295/role_data_model_access_462x333.png)
4. In Permissions tab, configure permissions for this role similarly to [Tenant Permissions](https://devnet.logianalytics.com/hc/en-us/articles/12528299938583-Tenant-Setup#Edit).
5. In Users tab, tick users in Assigned Users.

   > Assigning users is not available when Izenda is integrated inside an existing system.
   >
   > ![../_images/Role_Users_tab.png](https://devnet.logianalytics.com/hc/article_attachments/12528204682519/role_users_tab_462x308.png)
6. The Permission Summary tab is a read-only comparison of permissions among all roles of the current tenant.

   [![../_images/Role_Permission_Summary.png](https://devnet.logianalytics.com/hc/article_attachments/12528220558359/role_permission_summary_659x627.png)](https://devnet.logianalytics.com/hc/article_attachments/12528220545559/role_permission_summary.png)

   For example: a quick scan through this page ensures that mod002 role consists of User and Role setup permissions only while mod001 role consists of Data Model, Report and Dashboard permissions only.
7. Click the Save button at the top.
8. The new role will appear in the list in Middle Panel.
9. Click the Activate link next to the role in Middle Panel to make it active.

## Edit Role

1. Click the role in the list in Middle Panel.
2. The role’s information is displayed in the 4 tabs.
3. Edit the information.
4. Click the Save button at the top, then OK in the confirmation pop-up.

1. Untick the Active checkbox (☑) of the role in Middle Panel.
2. Click OK in the confirmation pop-up.
3. The role becomes inactive. Its users no longer have the permissions specified in this role.

Note :The Cancel button at the top will have no effect in this case.

## Delete Role

1. Click the Delete icon (x) of each role.
2. Click OK in the confirmation pop-up.
3. The role is deleted and removed from the list in Middle Panel.

Note: The Cancel button at the top will have no effect in this case.
