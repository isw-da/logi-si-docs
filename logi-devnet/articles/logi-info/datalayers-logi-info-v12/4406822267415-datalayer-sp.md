# DataLayer.SP

Developers working with SQL databases often prefer to use **Stored Procedures** rather than direct SQL statements in their applications for better performance and enhanced security.

The following topics introduce you to the DataLayer.SP element, which provides the capability of running stored procedures on SQL database servers:

* [DataLayer.SP Attributes](https://devnet.logianalytics.com/hc/en-us/articles/4406817318295-DataLayer-SP-Attributes)
* [Retrieving Data with DataLayer.SP](https://devnet.logianalytics.com/hc/en-us/articles/4406817320343-DataLayer-SP-Retrieving-Data-with-DataLayer-SP#Retrieving)
* [Stored Procedure Parameters](https://devnet.logianalytics.com/hc/en-us/articles/4406817321495-DataLayer-SP-Stored-Procedure-Parameters)
* [Writing Data to the Database](https://devnet.logianalytics.com/hc/en-us/articles/4406817323543-DataLayer-SP-Writing-Data-to-the-Database#Writing)
* [Using Studio's DataLayer Wizard](https://devnet.logianalytics.com/hc/en-us/articles/4406817322519-DataLayer-SP-Using-Studio-s-DataLayer-Wizard#Wizard)

![](https://devnet.logianalytics.com/hc/article_attachments/4417562936855/criticalicon.png)Stored procedures executed using this element *must* return a rowset. Ensure that they end with a **SELECT** statement.

Using Oracle? Connection strings to Oracle databases must include the statement "PLSQLRSet=1" in order to use stored procedures.
