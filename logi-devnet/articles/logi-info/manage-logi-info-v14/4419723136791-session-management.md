# Session Management

# Session Management

Developers may want to exert control over the time span of a user's
application session. This can be done by configuring an appropriate
session timeout period for applications and, in the case of embedded Logi
applications or components, may need to be synchronized with a parent
application. The application's session timeout should match the timeout
configuration of the application server.

[Session Variables](https://devnet.logianalytics.com/hc/en-us/articles/4419723341847-Session-Variables)
provides insight into how these variables are used and how Java and .NET
applications can share them. Logi Info also provides a
**Session Timeout** element, described in
[Managing Session Timeout](https://devnet.logianalytics.com/hc/en-us/articles/4419715459991-Managing-Session-Timeout-), to assist with management of the user session.
