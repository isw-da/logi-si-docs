# @Date Tokens

# @Date Tokens

The token identifiers listed in the table below all deal with dates. For example, @Date.Yesterday~ will provide yesterday's date.

Dates are expressed by default in the *yyyy-M-d* format, which does not include leading zeros for single-digit day or month values. Examples: April 1st = "2009-4-1", April 25th = "2009-4-25", October 31st= "2009-10-31".

If needed, you can override this default format by specifying a new format in the Globalization element's **Default Input Date Reformat** attribute. For more information on globalization, see [Internationalization and Localization](https://devnet.logianalytics.com/hc/en-us/articles/4419707658647-Internationalization-and-Localization-). The Default Input Date Reformat attribute ensures that dates entered by the user are formatted identically
to dates returned from @Date tokens. This is especially useful if you are using the @Date tokens with the Validation.Date element.

The Globalization element can be added in the \_Settings definition.

To work with the @Date.FiscalQuarter~ and @Date.FiscalYear~ tokens, use the Globalization element to set the first day of the fiscal year.

@Date.TodayUtc~ returns the UTC (Greenwich Mean Time) date.

The TimePeriodColumn element can also be used to parse specific date parts out of @Date token values. For more information on the TimePeriodColumn, see [Send Cell Phone SMS Messages](https://devnet.logianalytics.com/hc/en-us/articles/4419715458071-Send-Cell-Phone-SMS-Messages).

## New for 14.2 Customizing @Date Tokens

You can also customize the @Date token to retrieve relative dates for reports and report scheduling. This feature is available anywhere that you use existing @Date tokens.

1. Begin by double- clicking the **Customize…** option in the token value list:

![](https://devnet.logianalytics.com/hc/article_attachments/7522767563927/studio_select_customize.png)

The Customize Date Token dialog displays.

1. Enter a number in the first field:

![](https://devnet.logianalytics.com/hc/article_attachments/7522789074967/customize_date_token_dialog_619x196.png)

1. Then, select the **Date Section** drop-down and choose a **value**:

![](https://devnet.logianalytics.com/hc/article_attachments/7522803609111/image-20220711-195600_648x278.png)

1. Next, select the **Date Direction** drop-down and choose a **direction**:

![](https://devnet.logianalytics.com/hc/article_attachments/7522784156439/image-20220711-195655_640x203.png)

1. Select the **Date Edge** drop-down and select an **edge**:

![](https://devnet.logianalytics.com/hc/article_attachments/7522767604119/image-20220711-195757_639x205.png)

![](https://devnet.logianalytics.com/hc/article_attachments/7522725179159/noteicon.png) Depending on your selections, some options may not be available in other drop-downs. For example, if Date Section is set to "Hour", "On" is not available in this drop-down.

1. Select the **check box** to enable time input. Once enabled, enter the time (HH:mm). ![](https://devnet.logianalytics.com/hc/article_attachments/7522725179159/noteicon.png) Depending on your selections, the time input may be disabled.

1. When finished, select **OK**:

![](https://devnet.logianalytics.com/hc/article_attachments/7522775898135/select_ok_date_token_customize_647x204.png)

The @Date token value automatically populates, like below:

![](https://devnet.logianalytics.com/hc/article_attachments/7522803639575/data_token_manual_input_833x164.png)

You can also customize the @Date token by entering the value directly. This value is case sensitive.
