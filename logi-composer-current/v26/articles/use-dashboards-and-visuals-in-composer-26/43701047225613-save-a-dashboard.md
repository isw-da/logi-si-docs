# Save a Dashboard

When you create a new dashboard, you can save it with no content in the [library](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701047640717-Use-the-Library-for-Dashboards). You are prompted to add a new visual or an existing visual, but you can add a rich text snippet or simply save the dashboard and return to edit it later. The dashboard is saved in the [library](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701047640717-Use-the-Library-for-Dashboards).

When you attempt to save changes to a dashboard, Composer automatically saves any changes to:

* local visuals
* rich text snippets
* filter snippets
* added and repositioned shared visuals with no unsaved data changes

If your dashboard includes shared visual gallery visuals, Composer checks to for unsaved data changes to the visuals. If there are unsaved changes, you can elect to save each affected visual when you save the dashboard.

**Note:** 
When you save a dashboard, only the last time bar field you were using and its range and playback configuration are saved.

The following factors affect your ability to save a dashboard and its visuals.

* You must log in as an Owner or Editor of a dashboard, or have the **Create Dashboards** [privilege](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701005611789-Group-Privilege-Reference).
* Shared visual gallery visuals are saved if you have **Create Visuals** (or **Administer Visuals**) [privilege](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701005611789-Group-Privilege-Reference).
* The visuals on the dashboard will only be saved if write [permission](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701184646285-About-Visual-Permissions) for the visual has been granted to the user, one of the user's groups, or the user's Composer account. See [About Visual Permissions](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701184646285-About-Visual-Permissions).

**Save a dashboard**

1. In the dashboard header, select the Save icon (![](https://logi-composer-v26.insightsoftware.com/hc/article_attachments/46243288768909)) on the [dashboard icon bar](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701047597837-Use-the-Dashboard-Icon-Bars). (The Save As (![](https://logi-composer-v26.insightsoftware.com/hc/article_attachments/46242662918285)) icon allows you to make a [copy](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701076444557-Copy-a-Dashboard) of the dashboard.)

   The Save Options dialog appears.

   ![determine save options for your dashboard and unsaved visuals](https://logi-composer-v26.insightsoftware.com/hc/article_attachments/46242696583949 "save options work area - unsaved shared visuals")
2. In the **Name** box, enter a title for your dashboard. This is the name by which the dashboard will be saved.

   * If you want to save the dashboard using a different name, change it here. The original dashboard is renamed. See [Rename a Dashboard](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701076843149-Rename-a-Dashboard)
   * If you want to make a copy of a dashboard using a different name, see [Copy a Dashboard](https://logi-composer-v26.insightsoftware.com/hc/en-us/articles/43701076444557-Copy-a-Dashboard).
3. If you want to provide details about your dashboard, do this in the **Description** box. A maximum of 750 characters can be specified. Leading and trailing spaces are not allowed.
4. Review the list of shared visuals with unsaved changes on the **Existing Visuals** tab, if there are any.

   ![select the check box to save changes to unsaved visuals with changes](https://logi-composer-v26.insightsoftware.com/hc/article_attachments/46242665299725 "unsaved changes visuals dialog")

   Select the checkbox on this tab to save the changes to the visuals. Leave the checkbox cleared if you do not want to save the changes to the existing visuals. When you save them, the visuals are changed on every dashboard on which they are used. If you do not save the visuals, your changes will be discarded when you close the dashboard (a warning dialog displays first).
5. Select **Save** to save the dashboard or **Cancel** to cancel.
