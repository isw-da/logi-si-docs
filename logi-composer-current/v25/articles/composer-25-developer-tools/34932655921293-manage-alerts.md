# Manage Alerts

Use alerts to alert your end users when a metric reaches a specified threshold. Use the [Logi Composer user interface](https://logi-composer-v25.insightsoftware.com/hc/en-us/articles/34932680828941-Managing-Alerts) or API to manage your alert definitions. These definitions describe an alert condition, determine a schedule to evaluate the alert condition, and how notifications are handled when an alert condition is met. See[Managing Alerts](https://logi-composer-v25.insightsoftware.com/hc/en-us/articles/34932680828941-Managing-Alerts), [Create an Alert Definition](https://logi-composer-v25.insightsoftware.com/hc/en-us/articles/34932664892685-Create-an-Alert-Definition)  and [Alerts API](https://logi-composer-v25.insightsoftware.com/hc/en-us/articles/34932664791437-Alerts-API).

## Prerequisites

* Configure the mail service for Logi Composer.
* Configure [dashboard alert link redirection](https://logi-composer-v25.insightsoftware.com/hc/en-us/articles/34932665112589-Configure-Dashboard-Alert-Links).
* Ensure intended recipients have valid email addresses specified in their Logi Composer user definitions.
* Grant the group privilege **Administer Alerts** or **Create Alerts** to users who you want to enable to create or administer alerts. Administrative users are granted these privileges' by default. See [Group Privilege Reference](https://logi-composer-v25.insightsoftware.com/hc/en-us/articles/34932577846157-Group-Privilege-Reference).
* Grant users who receive alerts read access to the data sources used in alert conditions. If a user doesn’t have read permission for a data source will not receive any alert notifications. The corresponding alert messages also appear in the scheduler report and the job scheduler. See [About Source Permissions](https://logi-composer-v25.insightsoftware.com/hc/en-us/articles/34932915596173-About-Source-Permissions).
